# 蜀安首页方案 A 设计评审整改快照

日期：2026-06-27

范围：`src/prototypes/system-replica-base/` 默认页 `shuan-home-command`（蜀安首页方案 A）

依据：`.axhub/make/design-audits/system-replica-base-2026-06-27/design-review.md`

## 本次整改目标

把当前首页从偏“展示大屏”进一步收敛为“值守人员可快速判断和操作的监测工作台”。优先解决以下问题：

1. 告警优先级不够突出，值守人员第一动作不明确。
2. 地图点位缺少风险语义，视觉重心大但决策信息偏弱。
3. 提醒通知偏信息展示，缺少明确责任动作。
4. 小屏宽度下右侧关键告警区域容易被静默裁切。

## 已落实的页面决策

### 1. 新增“当前最紧急事项”

在顶部导航下方新增全局紧急事项条，聚焦四类值守优先信息：

- 红色告警：9 条，突出达州、成都、宜宾的红色告警分布。
- 待处置任务：21 条，明确需要派单处理。
- 高风险点位：12 处，提示需要现场复核与隐患确认。
- 最高风险地区：达州市，作为当前优先关注区域。

每一项都带有动作按钮，如“去核查 / 去派单 / 去确认 / 查看”，让用户看到后能判断下一步。

### 2. 地图点位按风险等级分层

四川地图点位从单一蓝色改为风险语义分色：

- 红色：最高风险 / 红色告警聚集。
- 橙色：较高风险 / 待处置量较高。
- 黄色：需关注点位。
- 蓝色：普通提示。
- 绿色：正常运行（图例预留）。

点位同时显示“煤矿数”和“待处置告警数”，并补充图例，避免地图只作为装饰。

### 3. 提醒通知改为任务卡

右上“提醒通知”由横向信息条调整为任务卡表达：

- 生产证照核验：3 家过期，动作“去核验”。
- 重大灾害预警：2 条，动作“去派单”。

任务卡保留提醒内容，但更强调责任动作。

### 4. 可读性和适配收敛

CSS 追加专门的 2026-06-27 优化层：

- 提升紧急指标数字层级。
- 为红 / 橙 / 黄 / 蓝建立稳定色彩语义。
- 为地图点位增加风险闪烁和待处置标签。
- 增加 `max-width: 1500px`、`max-width: 1400px` 的布局收敛规则，减少低宽屏裁切风险。
