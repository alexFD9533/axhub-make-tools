# 06-11短信管理模块迭代需求

## 一、迭代背景

短信发送记录查询页原仅支持本单位维度，需扩展至其他单位；短信配置模块需新增其他单位通讯录查看能力。

## 二、迭代目标

1. 短信发送记录查询页增加"本单位/其他单位"页签切换，其他单位维度增加"发送单位"查询与展示。
2. 短信配置菜单下新增"其他单位通讯录"列表页（左树右表），仅保留查看能力。
3. 其他单位通讯录查看页为纯只读，分基本信息、短信接收规则、接收企业三个可收起模块。
4. 保持现有平台后台样式与交互习惯。

## 三、迭代范围

仅适用达州平台。

### 3.1 涉及页面

| 页面 | 迭代类型 | 说明 |
|------|---------|------|
| 短信发送记录查询 | 改版 | 增加本单位/其他单位页签，其他单位增加发送单位字段 |
| 其他单位通讯录 | 新增 | 短信配置菜单下新增通讯录查看列表页 |

## 四、需求说明

### 4.1 短信发送记录查询页改版

查询区上方新增"本单位/其他单位"页签，默认展示"本单位"。

#### 4.1.3 本单位页签

沿用原页面查询和表格结构，保留原有字段与操作习惯。

#### 4.1.4 其他单位页签

查询区新增"发送单位"下拉选择（大竹县、达川区、宣汉县等），表格新增"发送单位"列（置于序号之后）。切换页签时查询区与表格字段动态变化，切回本页签后恢复原结构。

### 4.2 新增其他单位通讯录页

页面名称：其他单位通讯录；入口：短信配置菜单下，与"企业接收人员""政府接收人员"同级。

采用"左树右表"结构：
- 左侧：单位通讯录树，支持多个根节点（达州市、大竹县、宣汉县、渠县、万源市应急管理局）。
- 右侧：人员通讯录表，支持查询和查看。

列表字段：序号、所属单位、姓名、职务、手机号码、短信接收规则数、是否接收短信、是否设置、接收企业（家）、操作（仅查看）。

明确不展示：数据权限、新增、编辑、删除、批量配置。

### 4.3 其他单位通讯录查看页

页面标题：详情。视觉上参考原"政府接收人员"配置页，交互上纯只读。

#### 4.3.2 页面分组

拆分为基本信息、短信接收规则、接收企业三个独立分组，每组右侧带收起/展开箭头。

#### 4.3.3 基本信息

字段：姓名、职务、手机号码、是否接收短信、所属单位、所属部门、短信接收规则数、接收企业数。所有字段只读，不展示保存/取消/编辑/数据权限。

#### 4.3.4 短信接收规则

改为独立表格展示，字段：规则名称、是否为煤矿规则、状态。表格只读，无编辑/删除语义。

#### 4.3.5 接收企业

独立模块表格只读展示，字段：序号、区县、企业名称。不展示勾选列、企业选择框、保存/取消、编辑/删除。

## 五、交互与视觉要求

### 5.1 视觉基调

沿用现有平台后台样式，不引入新视觉体系，查询区/页签/表格/左侧树/分组标题/关闭按钮与系统底座保持一致。

### 5.2 交互要求

| 交互 | 要求 |
|------|------|
| 页签切换 | 短信发送记录查询页可在本单位/其他单位之间切换 |
| 查看跳转 | 其他单位通讯录列表点击"查看"进入查看页 |
| 分组折叠 | 三个模块右侧箭头可收起/展开 |
| 只读约束 | 查看页不得出现保存、取消、编辑、删除、勾选等操作语义 |

## 六、验收标准

### 6.1 短信发送记录查询

默认展示本单位页签；其他单位页签可切换，出现"发送单位"查询字段与表格列，示例数据包含大竹县、达川区等。

### 6.2 其他单位通讯录列表

短信配置菜单下可进入；左侧单位树支持多根节点；表格含"短信接收规则数"，不含"数据权限"；操作列仅"查看"。

### 6.3 其他单位通讯录查看页

点击"查看"进入子页面；标题为"详情"；右上角关闭按钮为系统一致的 X；拆分为三个可收起模块；短信接收规则表字段为规则名称/是否为煤矿规则/状态，无操作列；接收企业表无勾选列；页面整体只读。
