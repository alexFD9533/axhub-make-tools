# 蜀安首页 V3 二级下钻方案（2026-06-30）

## 一、方案目标

蜀安首页 V3 作为省级综合态势首页，一级页面只承担全省态势、重点风险和业务入口的总览职责。二级下钻用于承接具体业务查看、对象筛选和轻量处置判断，避免首页继续堆叠明细、表格和完整办理流程。

本方案明确首页各模块的二级下钻方式，并作为后续原型实现和验收依据。

入口心智规则：

- “更多”：进入该模块的独立二级页，用于看模块全貌；打非治违进入算法介绍页。
- 下划线：查看当前字段、卡片、任务或线索的局部明细，在当前页面弹出。
- 地图点位：进入地区维度驾驶舱，用于看市州监管态势。
- 顶部图标：进入跨模块工具页或打开全局消息弹层。
- 不下钻：纯展示、纯说明、辅助比例和装饰性指标不提供点击能力。

三类业务入口不得混用。若同一模块同时具备“更多”和下划线，“更多”只代表模块级页面，下划线只代表局部明细。

## 二、交互规则

### 1. 整体下钻

适用对象：模块本身具备完整业务专题价值，进入后可以形成一个独立页面。

表现规则：

- 在模块标题行右上角增加“更多”入口。
- “更多”使用文字按钮，不占用模块主体内容。
- 点击“更多”进入新页面，页面保持当前系统头部和平台风格。
- 二级页面标题应与模块业务一致，例如“煤矿生产状态监管”“数据资源接入健康监测”。
- 二级页面需要能独立撑满页面，不做只有表格或只有几个指标的空页面。

页面最小内容标准：

- 至少 4 个关键指标。
- 至少 2 个分析分区，例如趋势、分布、排行、分组、风险链条。
- 至少 8 到 10 条清单样例。
- 至少 1 个明确操作入口，例如查看矿井画像、进入线索详情、标记督办。

### 2. 单卡片下钻

适用对象：卡片只代表一个局部指标、一个分类、一个风险项或一条任务，不适合单独开完整页面。

表现规则：

- 可下钻的卡片标题、数字或字段增加下划线。
- 鼠标悬停时保持下划线，并显示可点击状态。
- 点击后在当前页面弹出轻量详情，不跳新页面。
- 弹出层优先使用右侧抽屉；需要展示证据链时可使用居中浮层。
- 弹出层必须支持关闭，关闭后回到原首页状态。

入口密度规则：

- 同一模块首屏第一批下划线入口不超过 5 个。
- 优先开放高风险、高待办、高频查看项。
- 其余可下钻项可以先保留为后续扩展，不在第一批全部开放点击。

### 3. 地图点位下钻

适用对象：地图上的市州聚合点。

表现规则：

- 点击市州聚合点进入“市州监管驾驶舱”。
- 点击结果必须携带市州参数。
- 默认没有明确市州参数时，进入“全省监管驾驶舱”或保留当前选中的市州。
- 市州驾驶舱必须提供返回首页入口。

### 4. 顶部图标入口

适用对象：视频、报警消息等不属于单个首页模块标题行的全局入口。

表现规则：

- 顶部视频图标进入“视频联网调度页”。
- 顶部报警图标打开“报警消息弹层”。
- 图标必须具备 tooltip 或可读标签，说明点击结果。
- 图标入口应有 hover、focus、active、disabled 状态。

### 5. 不下钻

适用对象：纯展示、纯说明、辅助比例、装饰性指标。

表现规则：

- 不加“更多”。
- 不加下划线。
- 不提供弹出层。
- 避免让用户误以为所有数字都可点击。

### 6. 导航与返回

- 二级页必须保留平台头部。
- 二级页应提供面包屑或“返回首页”按钮。
- 从“更多”进入的二级页，浏览器返回和页面内返回都应回到首页。
- 从地图点位进入的市州驾驶舱，返回后保留原首页状态。
- 弹出层关闭后，焦点回到触发入口，首页筛选条件不改变。
- 弹出层内如提供“进入详情页”，必须明确去向：所属二级页、三级算法明细页、单条详情、矿井画像或线索详情。

## 四、整体下钻页面规划

### 1. 市州监管驾驶舱

入口：地图市州聚合点。

页面定位：地区维度驾驶舱，不使用模块标题行“更多”。

页面内容：

- 市州监管总览：煤矿总数、生产矿井、风险矿井、今日预警、待处置事项。
- 市州地图：展示该市州内矿井点位和风险等级。
- 重点矿井：高风险矿井、连续预警矿井、整改临期矿井。
- 监管事项：日常监管、打非治违、灾害风险、数据异常。
- 处置清单：待办、超期、督办、已闭环。

落地补充：

- 市州参数来自地图点位。
- 默认市州可取当前选中市州；无选中时进入全省态势。
- 点击矿井进入矿井画像或矿井详情。
- 返回后保留首页原筛选和地图状态。

### 2. 煤矿生产状态监管

入口：煤矿生产状态统计模块标题行“更多”。

页面内容：

- 状态分布：生产、建设、隐患整改、一停四不停、井口封闭。
- 区域分布：各市州不同状态矿井数量。
- 状态变化：新增生产、转停产、整改完成、关闭退出。
- 异常提醒：停产矿井异常活动、整改临期、封闭矿井复发线索。
- 矿井状态清单：矿井、地区、当前状态、风险等级、更新时间、操作入口。

### 3. 生产运行监测

入口：煤矿生产情况模块标题行“更多”。

页面内容：

- 产量总览：年度产量、本月产量、日产量趋势。
- 作业面情况：采煤工作面、掘进工作面、停用工作面、异常工作面。
- 生产强度：产量突增、夜间生产、超计划生产、停产仍有生产迹象。
- 关联风险：生产强度与瓦斯、水害、人员超限的关联提示。
- 异常矿井清单：矿井、异常类型、关联数据、当前状态。

### 4. 人员安全监管

入口：从业人员情况模块标题行“更多”。

页面内容：

- 人员总览：从业人员、当前井下人员、重点岗位、特种作业人员。
- 井下分布：按市州、矿井、区域、工作面展示人员分布。
- 异常预警：超员、滞留、脱岗、求救、定位离线。
- 关键岗位到岗：矿长带班、安监员、瓦检员、特种作业人员。
- 异常人员清单：人员、矿井、异常类型、持续时间、处置状态。

### 5. 数据资源接入健康监测

入口：数据汇聚情况模块标题行“更多”。

页面内容：

- 接入总览：已接入数据项、异常数据项、离线接口、延迟接口。
- 数据分类健康度：基础数据、人员数据、灾害数据、监测数据、生产数据、事故保障数据。
- 来源单位矩阵：省级平台、市州系统、矿端系统、第三方系统。
- 数据质量问题：缺字段、超时未更新、格式异常、重复上报、数据冲突。
- 业务影响：影响日常监管、打非治违、风险预警、矿井画像的具体事项。

### 6. 瓦斯灾害风险研判

入口：煤矿瓦斯灾害情况模块标题行“更多”。

页面内容：

- 瓦斯风险总览：高瓦斯、低瓦斯、突出矿井、报警数量、未闭环报警。
- 监测趋势：瓦斯浓度、风速、负压、抽采数据。
- 风险矿井排行：报警频次、持续时间、处置超期、历史隐患。
- 风险链条：瓦斯异常关联通风、人员、作业面、危险作业。
- 报警处置台账：报警时间、点位、级别、确认情况、处置措施。

### 7. 水害防治专题

入口：煤矿水害情况模块标题行“更多”。

页面内容：

- 水害类型分布：简单、中等、复杂、极复杂。
- 重点隐患：老空水、承压水、采空区积水、断层导水。
- 监测预警：涌水量、水位变化、降雨影响、周边矿井联动风险。
- 探放水管理：计划、执行、验收、未完成事项。
- 治理工程台账：工程名称、矿井、进度、风险等级、责任单位。

### 8. 日常监管二级页

入口：日常监管模块标题行“更多”。

页面定位：模块级二级页，不做完整子系统。

边界规则：

- 只承载今日监管态势、风险分组、重点清单和轻量操作入口。
- 不新增多级菜单。
- 不承载复杂组合查询。
- 不做完整办理闭环。

页面内容：

- 今日监管事项：总数、待处置、超期、已闭环。
- 风险分级：红、橙、黄、蓝风险数量和占比。
- 监管类型：环境监测、人员定位、危险作业、证照提醒。
- 重点清单：今日最需要关注的 8 到 10 条事项。
- 操作入口：查看详情、标记督办、进入矿井画像。

### 9. 打非治违算法介绍页

入口：打非治违模块标题行“更多”。

页面定位：算法总览型二级页，不做完整执法平台。页面用于向监管用户说明打非治违算法体系、当前预警态势、覆盖煤矿范围和算法评分结果，并为重点算法进入三级分析明细页提供入口。

边界规则：

- 承载算法种类、预警数量、变化趋势、覆盖煤矿、评分分布和重点算法入口。
- 不承接完整线索办理流程。
- 不新增完整执法流程。
- 不做复杂案件管理。
- 不替代单条线索详情页；三级页只展示算法分析思路、样例证据链和矿井评分解释。

页面内容：

- 算法种类：按瓦斯、人、风、煤、电五类组织算法，展示每类算法数量、已启用状态、数据覆盖范围。
- 预警数量：展示今日预警、本月预警、待核查、较高风险矿井数量。
- 变化趋势：按日或按周展示 6 月以来预警数量、评分均值、较高风险矿井数量变化。
- 覆盖煤矿：展示覆盖煤矿数、覆盖率、最近数据时间、数据缺口。
- 煤矿评分：按矿井展示综合评分、算法命中项、风险等级、评分变化、是否进入三级分析。
- 重点算法入口：瓦斯、人、风、煤、电至少各提供 1 个三级明细页入口。
- 数据准备提示：标记算法所需数据来源、是否已接入、是否需要人工导入或离线分析。

二级页推荐布局：

- 顶部总览：算法总数、预警总数、覆盖煤矿数、较高风险矿井数。
- 算法分类区：瓦斯、人、风、煤、电五类算法卡片，展示启用算法、预警数量、评分均值和数据状态。
- 趋势区：展示 6 月以来预警数量和评分变化趋势，支持按算法类别切换。
- 覆盖煤矿及评分区：列表展示矿井名称、市州、覆盖算法、综合评分、命中算法、最近更新时间。
- 重点算法区：展示 5 类重点算法的“查看分析思路”入口，进入三级页。

### 10. 重点算法三级页面规划

三级页定位：解释重点算法如何形成预警或评分，让用户看懂分析思路、数据来源、样例矿井、关键证据和下一步核查方向。三级页可以从算法介绍页的算法卡片进入，也可以从覆盖煤矿及评分列表进入；不要求必须从预警下钻进入。

三级页通用内容：

- 算法说明：算法名称、所属类别、识别目标、适用场景。
- 数据来源：列出需要采集或导入的数据项、时间范围、来源系统、当前接入状态。
- 分析思路：用步骤化方式展示算法如何比对、计算、筛选和评分。
- 样例矿井：展示至少 1 家矿井的样例分析过程和评分解释。
- 证据链：展示关键指标、趋势图、数据差异、视频或离线分析结论。
- 输出结果：展示预警等级、可信度、评分、建议核查动作。
- 状态说明：覆盖数据缺失、离线分析中、无权限、评分不足以形成预警等状态。

## 六、样式与可访问性规则

### 1. “更多”入口

- 固定在模块标题行右上角。
- 文案统一为“更多”。
- 不使用“查看全部”“进入详情”“下钻”等多个不同文案。
- 样式应弱于主指标，强于普通说明文字。
- 使用真实 button 或 link 语义。
- 可读标签建议为“查看更多：模块名称”。
- 必须具备 hover、focus、active、disabled 状态。

### 2. 下划线入口

- 只给可点击字段加下划线，不给整块无差别加下划线。
- 下划线字段应保持原有字号和颜色体系。
- 悬停时可以增加亮色或轻微背景，但不改变布局。
- 已禁用或无明细的数据不加下划线。
- 不能只靠颜色表达可点击，必须同时具备下划线、指针样式和键盘焦点态。
- 点击目标区域应尽量达到 44 x 44 CSS px；文字较短时需要增加内边距。

### 3. 弹出层

- 日常监管类弹出层优先使用右侧抽屉。
- 打非治违类弹出层优先使用右侧抽屉；证据链较复杂时可使用居中浮层。
- 弹出层内容不超过当前页面高度，超出时内部滚动。
- 弹出层关闭后不改变首页筛选状态。
- 支持关闭按钮和 Esc 关闭。
- 打开弹层后焦点进入弹层，关闭前不应继续 Tab 到首页背后的内容。
- 关闭后焦点回到触发入口。
- 背景内容不可操作。

### 4. 顶部图标

- 视频、报警等顶部图标必须有可读标签。
- 图标点击目标应满足 44 x 44 CSS px。
- 图标 hover、focus、active、disabled 状态应与模块入口保持同一套视觉语言。

## 七、权限与状态

角色口径：

- 省级监管用户：可看全省、市州和矿井聚合信息，可进入市州驾驶舱。
- 市州监管用户：默认看本辖区数据，可进入本辖区驾驶舱。
- 矿端或企业端用户：如后续开放，仅看本矿井相关数据，不展示跨地区敏感明细。

状态口径：

| 状态    | 首页入口表现         | 二级页 / 弹层表现          |
| ----- | -------------- | ------------------- |
| 正常有数据 | 显示“更多”或下划线     | 展示指标、分析区、清单和操作入口。   |
| 无数据   | 不显示下划线，或显示普通文本 | 展示空状态和数据更新时间。       |
| 无权限   | 入口置灰或隐藏敏感入口    | 展示权限提示，隐藏敏感字段和操作按钮。 |
| 加载中   | 入口点击后显示加载反馈    | 显示骨架屏或加载状态。         |
| 加载失败  | 入口保持可重试        | 显示失败原因和重试入口。        |
| 禁用    | 不显示下划线或显示禁用态   | 不打开弹层，必要时显示禁用说明。    |

| 日常监管二级页   | 今日事项、风险分级、监管类型、重点清单、操作入口。           |
| 打非治违算法介绍页 | 算法种类、预警数量、变化趋势、覆盖煤矿、煤矿评分、重点算法三级页入口。 |
| 重点算法三级页   | 算法说明、数据来源、分析思路、样例矿井、证据链、输出结果、状态说明。  |
| 其他整体下钻页   | 4 个关键指标、2 个分析分区、8 到 10 条清单、1 个操作入口。 |

### 3. 弹层验收

- 弹层打开后焦点进入弹层。
- 弹层支持关闭按钮和 Esc 关闭。
- 弹层关闭后焦点回到触发入口。
- 弹层内部超出高度时内部滚动。
- 背景内容不可操作。
- 弹层提供加载中、无数据、加载失败、无权限状态。
- 弹层不承接完整办理，只提供轻量明细和二次入口。

### 4. 响应式与视觉验收

- 桌面至少验证 `1920 x 1080`、`1440 x 900`、`1366 x 768`。
- 至少补充一个窄屏或 200% 缩放场景。
- 页面无明显遮挡、重叠和横向溢出。
- “更多”、图标和下划线入口在 hover、focus、active、disabled 状态下不改变布局。
- 长标题、长矿井名称、长证据标签不得挤压按钮或覆盖后续内容。

### 5. 权限与异常验收

| 场景    | 通过标准                               |
| ----- | ---------------------------------- |
| 无数据   | 不误导用户点击；二级页或弹层展示空状态。               |
| 无权限   | 入口置灰或隐藏敏感操作；页面提示权限限制。              |
| 加载失败  | 显示失败状态和重试入口。                       |
| 浏览器返回 | 从二级页返回首页；首页状态保持。                   |
| 页面内返回 | 从二级页返回首页；首页状态保持。                   |
| 弹层内跳转 | 明确进入二级页、三级算法明细页、单条详情或矿井画像，不出现未知跳转。 |

## 十、后续开展方式

### 1. 先完成页面口径对齐

本轮沟通先确认页面层级和目标，不先进入视觉细节：

- 一级首页：只保留全省态势、重点风险和模块入口，不堆叠算法明细。
- 二级页面：承接模块总览。打非治违二级页明确为“算法介绍页”，说明算法体系和煤矿评分。
- 三级页面：承接重点算法分析思路。瓦斯、人、风、煤、电至少各 1 个算法三级页。
- 弹层：只用于轻量明细，不替代算法介绍页和三级算法页。

### 2. 再完成数据清单对齐

打非治违算法页需要先确认可用于演示的数据口径：

- 时间范围：默认从 2026 年 6 月开始组织趋势和样例数据。
- 基础对象：煤矿清单、市州、生产状态、评分、算法命中项、最近更新时间。
- 瓦斯数据：安技中心导入数据、瓦斯监测、产量、抽采、通风相关数据。
- 人员数据：精准人员定位、读卡分站、视频点、人员出入时段、离线录像分析结果。
- 风类数据：采掘工程、密闭资料、风速、总回风量、用风点、断面数据。
- 煤类数据：四表两图、采掘巷修进尺、工作面数量、生产班次、皮带秤、煤车视频分析结果。
- 电类数据：四表两图、产量、采掘巷修数量、进尺、重大设备功率、运行时长、电表采集数据。

### 3. 分阶段实现

第一阶段先完成可讲清楚的骨架：

1. 首页打非治违“更多”进入算法介绍页。
2. 算法介绍页展示五类算法、预警数量、趋势、覆盖煤矿和评分列表。
3. 三级页先按静态样例数据实现 5 类重点算法：瓦斯涌出量不匹配、疑似未带卡或人员停留异常、风速异常变化或风量不匹配、煤量变化分析、吨煤用电变化分析。
4. 每个三级页都能讲清楚“用了什么数据、怎么分析、为什么给出这个评分或预警”。

第二阶段补齐可验证的证据链：

1. 补充样例矿井数据、趋势图、对比表和评分解释。
2. 对离线视频分析、人工造数、安技中心导入等数据状态做明确标识。
3. 增加从煤矿评分列表进入对应算法三级页的路径。
4. 增加无数据、数据待导入、离线分析中、无权限等状态。

第三阶段再做体验细化：

1. 调整页面视觉层级，让二级页更像算法总览，三级页更像分析说明。
2. 补齐返回路径、面包屑、hover/focus 状态和长文本适配。
3. 用代表性矿井完成端到端演示：首页入口、算法介绍页、煤矿评分、算法三级页、返回。

### 4. 每轮验收方式

- 产品口径验收：页面层级是否清楚，是否避免把打非治违继续做成弹层集合。
- 数据口径验收：每个算法是否写清数据来源、时间范围、导入方式和缺口。
- 页面内容验收：二级页是否覆盖算法种类、预警数量、趋势、覆盖煤矿和评分。
- 分析思路验收：三级页是否能让用户看懂算法为什么命中、评分如何形成、下一步如何核查。
- 原型体验验收：入口、返回、三级跳转、空状态、无权限状态是否可用。
