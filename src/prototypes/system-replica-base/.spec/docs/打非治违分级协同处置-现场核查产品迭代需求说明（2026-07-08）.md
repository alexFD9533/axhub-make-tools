# 打非治违分级协同处置-现场核查产品迭代需求说明（2026-07-08）

## 一、迭代背景

蜀安首页 Av3 已经形成打非治违“算法线索 -> 专项整治 -> 分级处置”的一级入口结构，但“县级走访 / 现场核查”仍停留在首页卡片表达，缺少承接执行现场的二级页面。当前需要补齐现场核查执行页，用于承接县级核查组的签收、入场、取证、认定和结果回传。

## 二、迭代目标

- 在 Av3 drilldown 覆盖层内新增“现场核查 / 县级走访”页面。
- 让页面首屏直接表达任务状态、责任组和下一步流向。
- 让开发侧明确任务表、执行摘要、证据区和结论区的字段口径。
- 保持首页全局顶层壳层不变，只新增内容区实现与最小接入。

## 三、迭代范围

本次迭代适用于 `src/prototypes/system-replica-base/` 下钻体系。

### 3.1 涉及页面

| 页面 | 迭代类型 | 说明 |
| ---- | -------- | ---- |

## 四、需求说明

### 4.1 现场核查 / 县级走访

#### 4.1.1 页面定位

- 页面定位：现场核查执行页。
- 用户角色：县级联合核查组、县局值班人员、市级督办人员。
- 页面目标：回答“谁去查、查了什么、现场如何认定、下一步流向哪”。

#### 4.1.2 页面结构

1. 页面头部
   - 左侧：页面标题 `现场核查 / 县级走访` 与一句业务说明。
   - 右侧：退出按钮，返回首页 Av3。
   - 不新增顶部菜单、系统导航或其他全局壳层。
2. toolbox
   - 时间范围、所属区域、专项类型、任务状态。
   - 搜索框支持按煤矿、线索、专项、责任组检索。
   - 刷新任务、批量催办按钮只做静态表达，不接真实动作。
3. 顶部指标
   - 待签收
   - 待核查
   - 核查中
   - 已回传
   - 超时任务
4. 主表格
   - 煤矿名称
   - 线索来源
   - 所属专项
   - 派发时间
   - 截止时间
   - 当前负责人
   - 状态
5. 执行侧栏
   - 当前执行摘要：核查组、负责人、同行人员、核查对象、线索摘要。
   - 下一步流向：6 类结论标签 + 推荐流向说明 + 流转链路提示。
6. 底部证据与认定区
   - 核查重点清单
   - 现场记录
   - 照片视频
   - GPS / 轨迹与到离场

#### 4.1.3 字段与交互

1. 任务清单
   - 默认展示 5 条静态样例任务。
   - 点击行切换右侧执行摘要和底部证据区。
   - 状态标签口径：`待签收 / 待核查 / 核查中 / 已回传 / 超时任务`。
2. 执行摘要
   - 责任主体口径使用“核查组 / 当前负责人 / 同行人员 / 核查对象 / 线索摘要”。
   - 不展开真实通讯录、地图或轨迹详情。
3. 证据区
   - 核查重点清单按高 / 中 / 低优先级展示。
   - 现场记录按时间线展示关键过程语句。
   - 照片视频只展示计数，不做上传或预览弹层。
   - GPS / 轨迹区展示轨迹摘要、到场时间、离场时间、GPS 状态。
4. 认定流向
   - 固定 6 个结论标签：属实、部分属实、不属实、转整改、转市级研判、建议挂牌督办。
   - 每条任务提供一个推荐流向说明，作为县级核查后的下一步承接建议。

## 五、交互与视觉要求

- 只允许实现 `drill-stage` 内容区。
- 页面头部只保留标题和退出按钮，避免把筛选工具提升为全局 titlebar。
- 继续使用 deep-blue cockpit 视觉语汇：深蓝底、冷色描边、亮色状态标签、卡片式信息分区。
- 表格、证据卡和结论标签都需要保持与 `system-replica-base` 现有 `.drill-*` / 首页 Av3 风格一致。
- 首页“县级走访”卡片补最小点击接入，不要求本轮把全部分级处置卡片都改为 drilldown 页面。

## 六、验收标准

- 通过 `#page=shuan-home-command-v3-illegal-disposal-county-inspection` 可直接打开页面。
- 页面不复制系统顶栏、不新增全局导航。
- 页面首屏能看到 5 个指标、任务清单和执行摘要。
- 明确展示结论流向 6 类选项。
- 首页 Av3 的“县级走访”卡片可进入该页面。
- 执行 `$env:ENTRY_KEY='prototypes/system-replica-base'; npx vite build` 构建通过。
