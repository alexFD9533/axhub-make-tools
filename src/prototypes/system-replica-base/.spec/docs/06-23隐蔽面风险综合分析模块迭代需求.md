# 隐蔽面风险综合分析模块迭代需求说明（2026-06-23）

## 一、调整背景

隐蔽面风险综合分析模块的业务主线调整为“算法生成线索、线索基础加权、关联增强加权、综合风险评分、任务落实核查、结果回流修正煤矿风险等级”。线索不作为独立处置对象，风险也不作为独立事件对象。系统最终输出是煤矿当前存在隐蔽工作面可能性的风险分和风险等级。

## 二、本次原型范围

### 1. 风险总览一张图

本次将风险总览页调整为监管负责人查看区域风险、算法线索态势和重点煤矿的全局入口。

页面应包含：

- 筛选区：统计周期、区域、风险等级。跟踪状态、可疑点类型不作为一张图主筛选项。
- 顶部指标：煤矿总数、高风险煤矿、中风险煤矿、今日新增线索、参与评分线索、待核查高风险煤矿、逾期核查任务。
- 算法线索态势：按管住瓦斯、管住风、管住人、管住煤、管住电展示线索数、参与评分线索数和基础分贡献。
- 区域风险态势：按区县展示高/中/低风险煤矿分布和环比变化。
- 关联增强统计：展示时间关联、空间关联、对象关联、类型关联、状态冲突、历史关联、数据质量关联等命中情况。
- 风险等级变化趋势：展示近 7 天高/中/低风险煤矿数量变化，并可补充线索基础分、关联增强分变化。
- 高风险煤矿排行：展示煤矿、区域、风险分、风险等级、线索基础分、关联增强分、参与评分线索数、主要关联增强项、最近核查状态和环比。
- 核查任务与风险回流：展示待派发、待办理、专项排查中、结果已回流、持续跟踪、逾期任务。

#### 总览筛选口径

风险总览一张图不是线索检索页，不使用可疑点类型和跟踪状态作为主筛选。算法线索态势、关联增强统计可作为解释入口或下钻入口，但不应把全局态势切碎成变化不明显的筛选结果。

### 2. 煤矿风险画像

新增煤矿风险画像页，作为从总览进入单矿风险解释和任务落实的主页面。画像页围绕评分构成展开，回答“这家矿为什么高、高在哪里、下一步怎么查”。

页面应包含：

- 画像头部：煤矿名称、综合风险分、风险等级、区域排名、当前有效线索数、参与评分线索数、最近评分时间、最近核查状态和回流结论。
- 评分构成：线索基础分、关联增强分、回流修正分。
- 算法线索贡献：按管住瓦斯、管住风、管住人、管住煤、管住电展示线索数、参与评分线索数、基础分贡献，并可展开到算法维度。
- 关键线索：列出贡献较高或需要重点核查的算法线索，包含来源算法、发生时间、区域/对象、等级、有效性、基础分和证据材料。
- 关联增强分析：展示时间、空间、对象、类型、状态、历史、数据质量等关联维度命中情况、增强分和涉及线索。
- 风险解释与核查要点：根据主要线索和关联增强项生成检查问题、重点区域、重点对象和重点证据。
- 最近任务：展示专项排查、日常检查补充核查等任务状态。
- 风险回流判断：展示核查结果如何影响线索有效性、评分构成、风险等级维持/升高/降级/解除跟踪。
- 检查建议：列出现场核查和任务反馈时需要落实的重点。

## 三、业务口径

1. 算法模型生成线索，一个算法对同一煤矿可以生成多条线索。
2. 线索是评分输入和证据追溯对象，不作为独立处置对象。
3. 风险评分采用“线索基础分 + 关联增强分 + 回流修正分”口径。
4. 线索基础分由算法权重、线索等级、置信度、时效性、有效性、数据可信度等因素计算。
5. 关联增强分由时间、空间、对象、类型、状态、历史、数据质量等关联维度加权形成。
6. 可疑点不再作为总览筛选主线，降级为风险解释和核查要点的表达方式。
7. 任务围绕煤矿风险等级和核查要点创建，分为日常检查、重点核查、专项排查、复核确认等类型。
8. 任务完成不等于风险自动消除；核查结果必须回流到线索有效性和评分依据后，才可触发风险等级维持、升高、降级或解除跟踪。
9. 煤矿风险画像应回答“这家矿为什么高、高在哪里、下一步怎么查”。

## 四、验收标准

- 风险总览页统一采用“算法线索、线索基础分、关联增强分、煤矿风险等级、核查任务、结果回流”口径。
- 风险总览页不再把可疑点类型、跟踪状态作为主筛选项。
- 高风险煤矿排行可进入煤矿风险画像。
- 煤矿风险画像能展示风险等级、评分构成、算法线索贡献、关键线索、关联增强分析、核查要点、核查任务和风险回流判断。
- 原型说明面板可切换到风险总览和煤矿风险画像，并能查看对应说明。
