# 蜀安首页V3产品迭代需求说明（2026-06-30）

## 一、概要

蜀安首页 V3 是省级煤矿安全监管驾驶舱首页。本次迭代围绕页面名称、地图煤矿聚合数量、日常监管业务层级、打非治违模块结构和原型批注锚点进行收敛，目标是在不增加新模块的前提下提升首页信息识别效率和交付可说明性。

## 二、干系人

| 角色 | 关注点 |
|---|---|
| 业务方 | 页面名称、煤矿聚合数量、监管业务顺序和重点风险表达是否符合汇报口径。 |
| 产品/原型评审 | 批注锚点、说明面板和需求文档是否能支撑后续评审、修改和交付。 |
| 前端开发 | 页面结构、文案、布局约束、验收标准和非范围边界是否清晰。 |

## 三、背景

蜀安首页 V3 用于呈现煤矿安全综合监控平台的省级监管首页。页面需要在一屏内表达煤矿总体状态、区域聚合分布、日常监管重点和打非治违线索处置情况。本次迭代围绕用户对页面名称、地图聚合数据、右侧模块层级和局部文案的反馈进行收敛，保持当前大屏式驾驶舱结构不扩展新模块。

## 四、目标

- 首页名称、地图聚合数量和右侧业务模块表达与最新评审意见一致。
- 日常监管中三类业务入口优先级更高，提醒类信息弱于业务入口。
- 打非治违模块去掉中间分析层后，仍保持从线索、专项整治到处置任务的业务链路。
- 原型批注锚点和 PRD 文档可被说明面板读取，方便后续按区域继续批注和验收。

关键结果：

| 指标 | 验收方式 |
|---|---|
| 批注定位完整 | 5 个核心区域均可通过 `data-annotation-id` 定位。 |
| 地图数据一致 | 12 个市州聚合数量与本需求表格一致。 |
| 一屏可读 | 1920×1080 常用展示尺寸下模块不重叠、不裁切。 |
| 构建通过 | `ENTRY_KEY=prototypes/system-replica-base npx vite build` 执行成功。 |

## 五、用户和使用场景

主要用户为省级煤矿安全监管人员、原型评审人员和后续前端开发人员。

- 监管人员在首页快速查看全省煤矿区域分布、日常监管重点和打非治违处置态势。
- 评审人员通过批注锚点定位页面关键区域，继续提出局部修改意见。
- 开发人员根据说明面板和 PRD 确认文案、顺序、布局和验收标准。

## 六、价值主张

- 对业务方：减少首页信息层级混杂，让“监管业务入口、风险表达、处置链路”更清楚。
- 对评审方：每个核心改动区域都有稳定锚点，后续批注不依赖模糊截图定位。
- 对开发方：需求文档直接说明页面结构、数据口径和不做范围，降低二次理解成本。

## 七、页面范围

页面路由：`shuan-home-command-v3`

本次调整范围限定在蜀安首页 V3：

- 顶部系统名称和视频入口。
- 左侧煤矿概况模块顺序。
- 中部四川地图煤矿聚合点。
- 右侧日常监管模块。
- 右侧打非治违模块。

不包含新增二级页面、弹窗、权限控制、真实接口联调和地图行政区边界重绘。

## 八、解决方案与页面结构要求

### 1. 顶部系统栏

- 系统名称显示为：`蜀·安煤矿多源异构监测系统`。
- 顶部工具区保留区域选择、视频入口、报警消息、用户、设置等入口。
- 视频入口以图标形式展示在报警消息铃铛附近，不新增说明文字。

### 2. 左侧煤矿概况

左侧模块顺序调整为：

1. 煤矿生产状态统计。
2. 煤矿生产情况。
3. 从业人员情况。
4. 数据汇聚情况。
5. 煤矿瓦斯灾害情况、煤矿水害情况。

瓦斯灾害和水害模块固定在左侧底部，作为自然灾害类补充信息，不再插入生产状态与生产运行模块之间。

### 3. 中部地图煤矿聚合

地图聚合点按市州展示煤矿数量：

| 市州 | 数量 |
|---|---:|
| 达州市 | 41 |
| 宜宾市 | 30 |
| 攀枝花市 | 18 |
| 乐山市 | 15 |
| 广安市 | 14 |
| 泸州市 | 13 |
| 广元市 | 11 |
| 雅安市 | 10 |
| 内江市 | 9 |
| 自贡市 | 7 |
| 凉山州 | 4 |
| 巴中市 | 2 |

地图点位按四川行政区相对方位校准。当前底图不包含精确市州边界，聚合点表达的是区域级聚合位置，不作为精确经纬度定位。

地图搜索支持按市州简称或名称匹配聚合点。输入匹配名称后，对应聚合点进入高亮状态，其他点位弱化。

### 4. 日常监管

- “危险作业报备 / 风险分级管控 / 隐患排查”作为监管业务入口行，展示在“证照到期提醒 / 重大灾害提醒”之前。
- 监管业务入口行高度加高，保证标题、数量和两个辅助指标可读。
- “风险分级”标题统一为“风险分级管控”。
- 风险分级管控卡片中的辅助指标显示为“高风险 38”和“动态 7”。
- “证照到期提醒 / 重大灾害提醒”保持轻量提醒样式，不作为主业务入口。

### 5. 打非治违

- 删除原“综合分析 / 多源线索交叉验证”中间行。
- 删除对应上下游连接线。
- 模块内保留三层结构：
  1. 五类智能算法线索：瓦斯、风、人、煤、电。
  2. 三类专项整治：隐蔽工作面专项整治、监控系统造假专项整治、隐瞒入井人数专项整治。
  3. 分级处置任务：煤矿整改、县级走访、市级研判、省挂牌督办。
- 删除中间行后，模块高度重新分配，避免底部处置区被拉伸或上方线索区拥挤。

- 页面顶部显示“蜀·安煤矿多源异构监测系统”。

- 左侧瓦斯灾害和水害模块位于左栏底部。
- 地图显示 12 个市州煤矿聚合点，数量与本需求表格一致。
- 日常监管中“风险分级管控”卡片显示“高风险 38”。
- 打非治违模块中不再出现“综合分析 / 多源线索交叉验证”文案和连接线。
- 页面在 1920×1080 常用展示尺寸下无页面级滚动，右侧模块内部不裁切、不重叠。
