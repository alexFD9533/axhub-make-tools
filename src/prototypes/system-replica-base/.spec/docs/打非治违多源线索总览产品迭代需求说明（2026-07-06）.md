# 打非治违多源线索总览产品迭代需求说明（2026-07-06）

## 一、概要

本次迭代针对蜀安首页 V3 下钻页“多源线索总览”进行重构收敛。页面不再承担线索详情和明细清单主承载任务，而是收敛为一个面向监管值守场景的算法线索总览工作台，重点回答“哪些算法在出线索、线索集中在哪些矿井、当前优先看什么”。

本次设计核心变化有两点：

- 左侧主区保留卡片化浏览，但改为“4 列算法卡片矩阵”，通过分类、搜索、排序、分页承载 50+ 算法。
- 原底部“线索动态”和“重点线索预览”合并为右侧轻量“重点线索预览”，给核心算法分布让出空间。

## 二、干系人

| 角色 | 关注点 |
|---|---|
| 监管值守人员 | 能否快速知道当前哪些算法、哪些矿井需要优先处理。 |
| 业务方 | 页面是否体现“多源线索总览”的业务定位，而不是线索详情页。 |
| 产品/原型评审 | 信息架构、字段口径、模块边界和排序逻辑是否清楚。 |
| 前端开发 | 页面布局、数据字段、交互规则和非范围边界是否足够明确。 |

## 三、背景

原页面更接近统计驾驶舱，左侧展示来源页面分布，右侧展示矿井排行和聚集提醒，底部再堆叠“线索动态”和“重点线索预览”。随着算法数量增加，这种结构存在三个问题：

1. 页面核心焦点不清，用户看不出“算法”才是主要分析对象。
2. 大卡片或大表格无法承载 50+ 算法，越加内容越拥挤。
3. 线索详情和明细清单占用过多首页空间，挤压了算法分布和矿井集中态势。

因此，本次迭代将页面定位明确为“算法线索总览工作台”，把完整明细和线索详情下沉到二级查看入口。

## 四、目标

- 页面主视角从“线索事件堆叠”调整为“算法分布 + 矿井聚集 + 重点预览”。
- 支持统一承载 50+ 算法，并通过分页解决数量问题。
- 让值守人员默认看到“最值得先看”的算法，而不是无差别平铺。
- 保持与蜀安首页 V3 现有深色工业监管风格一致，不单独新造一套视觉体系。

关键结果：

| 指标 | 验收方式 |
|---|---|
| 信息主轴清晰 | 首屏可明确识别“算法线索分布”为核心模块。 |
| 数量可承载 | 50+ 算法在不拥挤的前提下可通过分页完整浏览。 |
| 重点可识别 | 默认排序下，第一页展示最需关注算法。 |
| 布局稳定 | 1920×1080 常用展示尺寸下页面不重叠、不裁切。 |

## 五、用户和使用场景

主要用户为省、市、县三级监管值守人员和业务评审人员。

- 值守人员进入页面后，先看算法分布，判断当前主要风险由哪些算法触发。
- 结合右侧矿井集中排行，快速识别线索聚集矿井。
- 通过重点线索预览抽样查看高价值线索，决定是否进入二级详情。
- 评审和开发通过本需求文档确认页面定位、字段和交互边界。

## 六、价值主张

- 对业务方：页面更符合“多源线索总览”的业务语义，不再把首页做成详情页。
- 对值守人员：优先看到最重要算法和矿井，减少翻看低价值信息的成本。
- 对开发方：字段、分页、排序和模块边界清晰，便于实现与验收。

## 七、页面范围

页面路由：`shuan-home-command-v3-illegal-algorithms`

本次调整范围限定在该下钻页面内容区：

- 页面标题和筛选区。
- 分类总览卡片。
- 左侧“算法线索分布”主模块。
- 右侧“矿井线索集中排行”。
- 右侧“重点线索预览”。

不包含：

- 首页 Av3 主页面改造。
- 三层及以上新路由扩展。
- 真实接口接入、权限控制、导出联调。
- 完整线索办理流程、处置闭环详情页。

## 八、信息架构

页面信息主轴调整为：

1. 分类态势。
2. 算法线索分布。
3. 矿井线索集中排行。
4. 重点线索预览。

页面布局采用 70/30 双栏：

- 左侧 70%：算法线索分布工作区。
- 右侧 30%：上为矿井线索集中排行，下为重点线索预览。

## 九、页面结构与模块要求

### 1. 顶部筛选区

筛选区延续系统统一风格，建议包含：

- 时间范围
- 所属区域
- 所属矿井
- 状态

页面标题显示为：`多源线索总览`

副说明口径建议：`按分类和算法查看线索分布，快速定位线索集中矿井与重点异常。`

### 2. 分类总览区

按业务分类展示总量，不区分算法来源。分类包含：

- 全部
- 管住瓦斯
- 管住风
- 管住人
- 管住煤
- 管住电
- 其他

明确要求：

- 不展示“线索综合分析”分类。
- 不展示“四川添加算法 / 国家局算法”区分。

每个分类卡片建议展示：

- 算法数或命中算法数
- 线索数
- 高风险数
- 今日新增

### 3. 第二层核心模块：算法线索分布

该模块是本页核心模块，采用“工具区 + 4 列卡片矩阵”的表达方式，强调快速扫视和算法铺开，而不是精读分析。

#### 3.1 模块头部

左侧显示模块标题：`算法线索分布`

右侧显示模块级分页控件，建议样式：

`共 52 项   < 1 / 6 >`

要求：

- 分页控件放在模块头部右上角，不放页面底部。
- 分页属于模块内部控件，只控制算法卡片区。

#### 3.2 模块工具条

工具条建议包含：

- 分类页签：全部、管住瓦斯、管住风、管住人、管住煤、管住电、其他
- 搜索框：按算法名称搜索
- 排序下拉：高风险优先 / 线索数优先 / 命中矿井数优先 / 今日新增优先
- 开关：只看有线索
- 汇总指标：命中算法数、线索数、命中矿井数、高风险线索数

默认规则：

- 默认开启“只看有线索”。
- 默认排序为“高风险优先”。
- 分类切换后沿用当前排序方式。

#### 3.3 算法卡片字段

算法区建议采用 4 列卡片矩阵：

- 一行 4 个卡片
- 一屏展示 2 到 3 行
- 每页展示 8 到 12 个算法卡片

每张算法卡片仅展示以下字段：

| 字段 | 说明 |
|---|---|
| 算法名称 | 主标题字段 |
| 所属分类 | 例如管住瓦斯、管住风 |
| 线索数 | 该算法当前线索总数 |
| 命中矿井数 | 当前命中矿井数量 |
| 重点指标 | 高风险线索数或今日新增，二选一作为卡片重点值 |
| 风险强调 | 通过角标、色条或轻量标签表达高、中、低优先级 |
| 入口 | 查看按钮或详情入口 |

说明：

- 卡片中不展示长趋势图、长矿井列表、来源链路、处置链等重信息。
- 卡片目标是帮助用户快速扫视算法分布，不承担完整分析职责。
- 卡片与右侧排行、预览列表形成明显视觉区分。

#### 3.4 分页规则

分页用于解决算法数量承载问题，规则如下：

- 总算法量较大时启用分页。
- 每页展示 8 到 12 个算法卡片。
- 切换分类、搜索关键词、排序方式后，分页从第一页重新计算。
- “全部算法”不要求在首页一次性展示完。

### 4. 右侧模块一：矿井线索集中排行

该模块用于回答“线索主要集中在哪些矿井”。

建议字段：

- 排名
- 矿井名称
- 线索数
- 命中算法数
- 主要分类
- 风险等级或高风险数

要求：

- 保持为简洁排行结构，不展开完整明细。
- 默认展示 Top 5 到 Top 8。

### 5. 右侧模块二：重点线索预览

该模块由原“线索动态”和“重点线索预览”合并而成，仅承载少量重点线索抽样。

建议字段：

- 线索标题
- 所属矿井
- 来源算法
- 时间
- 风险等级
- 状态

要求：

- 默认展示 4 到 6 条重点线索。
- 以“预览”定位呈现，不替代完整详情页。
- 不再在本页保留大面积动态表格。

## 十、交互规则

1. 点击分类卡片或分类页签后，左侧算法卡片区、右侧矿井排行和重点线索预览同步切换到对应分类视角。
2. 搜索框仅作用于算法名称匹配，不检索矿井或线索标题。
3. 排序仅作用于左侧算法卡片区。
4. 分页仅作用于左侧算法卡片区，不影响右侧两个模块的数据范围。
5. 点击算法“查看”后进入该算法对应二级查看页或详情面板。
6. 点击重点线索后进入线索详情页或详情抽屉。

## 十一、数据口径

- 算法：统一算法清单，不区分四川补充算法和国家局算法。
- 命中算法数：当前筛选条件下，线索数大于 0 的算法数量。
- 线索数：当前筛选条件下的线索总数。
- 命中矿井数：当前筛选条件下至少命中 1 条线索的矿井数量。
- 高风险线索数：当前筛选条件下风险等级为高的线索数。
- 今日新增：按自然日统计新增线索数。

## 十二、视觉与实现约束

- 延续蜀安首页 V3 现有深色工业监管风格。
- 强调清晰、密度和实用，不做营销化大屏表现。
- 左侧主区优先使用卡片矩阵表达算法池，不增加大面积炫技图表。
- 卡片信息保持克制，避免重新膨胀成小型详情卡。
- 继续复用现有底座容器、筛选条、卡片、按钮和分页风格。

## 十三、发布边界

首版交付内容：

- 多源线索总览页面布局重构。
- 算法工作区的分类、搜索、排序、分页表达。
- 右侧矿井排行和重点线索预览重构。
- 配套设计快照和模块级需求文档。

后续版本再处理：

- 全量算法配置管理页。
- 完整线索详情、证据链和闭环办理页。
- 导出、联动筛选、跨页状态记忆。

## 十四、验收标准

- 页面标题显示为“多源线索总览”。
- 左侧“算法线索分布”为视觉主区，宽度明显大于右侧。
- 左侧模块头部右上角存在分页控件。
- 左侧算法卡片区支持分类、搜索、排序和“只看有线索”开关。
- 左侧算法区采用 4 列卡片矩阵，每页仅展示部分算法，不再全量平铺。
- 页面中不再出现“线索综合分析”“线索聚集研判”“四川添加算法 / 国家局算法”区分。
- 底部不再保留两个宽表格，“重点线索预览”收敛在右侧。
- 页面在 1920×1080 常用展示尺寸下不重叠、不裁切。
