# 打非治违分级协同处置-整改复核产品迭代需求说明（2026-07-08）
## 一、迭代背景
本次迭代面向蜀安首页 Av3 打非治违模块的“分级协同处置”链路，补齐第 1 个 drilldown 子页面“整改复核 / 县级协同处置”。

首页现状中，“分级处置任务”首卡已经承担“煤矿整改 / 县级复核”的业务语义，但缺少点击后的承接页，无法继续说明整改是否完成、县级是否通过、退回原因是否集中以及是否需要升级流转。

本次新增页面用于承接该入口，并明确口径：
- 首页入口继续保留原业务语义“煤矿整改 / 县级复核”。
- drilldown 页内统一采用新命名“整改复核 / 县级协同处置”。

## 二、迭代目标
- 为首页 Av3 打非治违模块补齐“煤矿整改 / 县级复核”首卡的直接下钻页面。
- 页面集中回答整改闭环与县级复核阶段的关键判断问题。
- 页面实现只覆盖 `drill-stage` 内容区，不改动全局顶栏和 `DrilldownFrame` 所有权。
- 同步补齐 route、说明文档和原型说明面板接线。

## 三、迭代范围
### 3.1 涉及页面

| 页面 | 迭代类型 | 说明 |
| ---- | -------- | ---- |
| `shuan-home-command-v3-illegal-disposal-rectification-review` | 新增 drilldown 页面 | 打非治违分级协同处置第 1 页 |
| 首页 Av3 打非治违“分级处置任务”首卡 | 最小接入 | 保留“煤矿整改 / 县级复核”文案并整卡点击直达新页面 |

### 3.2 不在本次范围
- 市级研判、省级督办等后续分级协同页面。
- 真实接口、导出联调、权限控制。
- 全局顶栏、首页整体布局或 `DrilldownFrame` 框架改造。

## 四、需求说明
### 4.1 页面定位
页面用于服务省、市、县三级监管值守与协同处置人员，重点回答：
1. 矿上改没改。
2. 县里过没过。
3. 为什么被退回。
4. 是否需要升级流转。

### 4.2 页面结构
页面采用“标题顶条 + 工具区 + 指标区 + 主工作区 + 底部支撑区 + 流程带”结构。

1. 标题顶条
   - 仅保留页面标题与退出按钮。
   - 不复用或复制系统全局顶栏。
2. 工具区
   - 放置统计周期、煤矿检索、状态筛选、县级视角、重点矿井等页内工具。
   - 导出动作也放在工具区。
3. 顶部指标
   - 待整改
   - 待复核
   - 复核退回
   - 已销号
   - 逾期事项
4. 焦点摘要区
   - 页面关注点摘要
   - 升级流转摘要卡
   - 整改前后对比
5. 主工作区
   - 左侧主表：整改复核任务台账
   - 右侧辅助区：复核结论、退回原因排行、升级流转判断
6. 底部支撑区
   - 附件材料
   - 复核动作提示
7. 闭环流程带
   - 问题下发
   - 企业整改
   - 提交复核
   - 县级复核
   - 销号归档

### 4.3 主表字段
主表字段固定为：
- 煤矿名称
- 问题来源
- 所属专项
- 整改期限
- 当前状态
- 责任人

状态建议保留：
- 整改中
- 待复核
- 复核退回
- 已销号
- 已逾期

### 4.4 入口接线要求
首页打非治违模块“分级处置任务”首卡需要满足：
- 保持原业务语义文案：
  - 标题：煤矿整改
  - 副标题：县级复核
- 整卡可点击，不是只点局部文案。
- 点击后直接切换到 route `shuan-home-command-v3-illegal-disposal-rectification-review`。

## 五、交互与视觉要求
- 页面只设计 `drill-stage` 内容区。
- 视觉继续沿用深蓝指挥舱语言，与 `system-replica-base` 一致。
- 标题顶条与工具区要明确分层，避免再次把筛选工具塞回全局顶栏。
- 1920 宽度下优先保证主表与右侧判断区同时可读。
- 小屏允许纵向堆叠，但不能出现面板重叠。

## 六、验收标准
- 直接访问 `#page=shuan-home-command-v3-illegal-disposal-rectification-review` 可打开页面。
- 首页 Av3 打非治违“分级处置任务”首卡整卡点击后，直接进入该页面。
- 页面顶部仅有标题与退出按钮。
- 页面具备待整改、待复核、复核退回、已销号、逾期事项 5 个核心指标。
- 主表、复核结论、退回原因排行、升级流转判断、附件材料、闭环流程带完整可见。
- 原型说明面板同步挂接本需求文档。
