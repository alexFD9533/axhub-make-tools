# 隐蔽工作面专项整治三级煤矿画像页产品需求初版（2026-07-08）

## 一、背景

当前蜀安首页 Av3 已明确存在以下页面层级：

1. 一级页面：蜀安首页 Av3
2. 二级页面：专题治理模块下钻后的“隐蔽工作面专项整治”专题页
3. 三级页面：在“隐蔽工作面专项整治”页中，点击煤矿清单里的某一座煤矿后进入的“煤矿画像页”

本次讨论和后续设计，针对的是第三级页面，不是二级专题页本身，也不是此前误入的“综合研判台”方向页面。

## 二、页面目标

该页不是单纯展示煤矿基础档案，也不是把线索明细简单铺开，而是作为“单矿风险解释页”，回答监管用户三个问题：

1. 这座煤矿为什么进入隐蔽工作面专项整治视野
2. 当前风险高在什么地方、由哪些信号和线索共同支撑
3. 下一步现场核查和任务处置应该重点看什么

因此，该页的核心定位是：

- 以“煤矿”为主对象
- 以“风险评分形成逻辑”为主主线
- 以“核查行动建议”为主输出

## 三、适用范围

### 3.1 涉及页面

| 页面 | 层级 | 说明 |
| ---- | ---- | ---- |
| 蜀安首页 Av3 | 一级 | 作为固定系统首页底稿，不承载本页业务内容 |
| 隐蔽工作面专项整治 | 二级 | 作为专题总览页，提供煤矿清单入口 |
| 隐蔽工作面专项整治-煤矿画像页 | 三级 | 本文档对应页面 |

### 3.2 入口与返回

- 入口：二级“隐蔽工作面专项整治”页面底部煤矿清单中的单条煤矿
- 返回：返回上一级“隐蔽工作面专项整治”专题页
- 不新增新的系统级顶部导航
- 不在本页承担跨专题切换职责

## 四、页面定位

### 4.1 页面身份

页面名称建议：

- 隐蔽工作面专项整治-煤矿画像

页面标题建议：

- 煤矿隐蔽工作面风险画像

页面副标题建议：

- 围绕单矿风险评分、关键线索、关联增强和核查建议，解释该矿当前隐蔽工作面风险状态

### 4.2 页面边界

本页需要承载：

- 单矿综合风险等级和评分解释
- 关键算法线索及其贡献
- 关联增强分析
- 风险解释与核查建议
- 最近任务与回流结果

本页不承载：

- 全省或全区域态势总览
- 专题级煤矿列表和排行
- 大批量线索检索页
- 完整执法办理流程
- 多专题并列工作台

## 五、页面结构

本页不再按“单屏驾驶舱”思路组织，也不建议继续使用“第一屏/第二屏/第三屏”这类说明式标题作为页面主结构。

建议改为“页内目录 + 内容主体”的三级画像页结构：

- 左侧为页内目录或段落导航
- 右侧为当前内容主体
- 页面主结构使用真正的业务模块标题，而不是说明式副标题

建议一级模块固定为三段：

1. 煤矿概况
2. 风险画像
3. 核查闭环

这样处理的目的，是让三级页从“专题总览压缩版”转为“单矿风险解释页”，同时兼顾以下几点：

- 比纯长页面更容易控制阅读节奏
- 比简单 tab 切换更像“画像档案页”
- 比单屏总盘更适合承载较多解释信息
- 后续实现时更容易稳定页面密度和模块边界

### 5.1 页面抬头区

该区位于页面主体顶部，属于“煤矿概况”模块的起点，用于先建立“当前看的是哪座矿、风险处于什么状态”的第一认知。

建议包含字段：

- 煤矿名称
- 所属地区
- 当前综合风险分
- 当前风险等级
- 区域内风险排名
- 当前有效线索数
- 参与评分线索数
- 最近评分时间
- 最近核查状态
- 最近回流结论

目标效果：

- 用户进入页面后，先知道“对象是谁、风险多高、现在处于什么阶段”

### 5.2 评分构成区

该区仍属于“煤矿概况”模块，是页面最先建立结论感的核心信息区，要讲清楚分数怎么来的，但要克制表达，不要把所有证据一次性堆满。

建议拆成 3 个分值来源：

1. 线索基础分
2. 关联增强分
3. 回流修正分

建议展示方式：

- 综合分值卡
- 分值构成条形拆分
- 各部分说明文案

每一部分都应能回答：

- 由什么形成
- 当前占比多少
- 为什么抬高或拉低综合风险

### 5.3 算法线索贡献区

该区进入“风险画像”模块，开始回答“为什么高”。

本区不做海量明细，而是按算法类别解释“哪些线索在支撑当前判断”。

建议按五类展示：

- 瓦斯
- 风
- 人
- 煤
- 电

每类建议展示：

- 命中线索数
- 参与评分线索数
- 贡献分值
- 代表性算法
- 当前异常摘要

必要时可展开到算法层，但默认不展开为长表，不采用大面积矩阵或复杂总览图。

### 5.4 关键线索区

该区仍属于“风险画像”模块。

本区聚焦最值得核查的少量高价值线索，而不是完整清单。

建议每条线索展示：

- 线索标题
- 来源算法
- 发生时间
- 涉及区域/对象
- 风险等级
- 可信度或有效性
- 基础贡献分
- 证据标签

建议控制在 4 到 6 条，体现“重点线索”，不做全量翻页。

### 5.5 关联增强分析区

该区属于“风险画像”模块后半段，是“综合分析理念”真正落地的地方，要说明为什么这些线索叠加后风险变高。

建议从以下维度表达：

- 时间关联：是否集中发生、是否持续反复
- 空间关联：是否集中在同一工作面、巷道或相邻区域
- 对象关联：是否涉及同一班组、同一区域设备、同一生产对象
- 类型关联：不同类别异常是否相互支撑
- 状态冲突：停产状态与人员/煤流/用电/通风信号是否矛盾
- 历史关联：是否存在历史复发或重复问题
- 数据质量关联：是否存在遮挡、缺失、异常平直等影响判断的情况

本区输出的不是抽象概念，而是：

- 命中了哪些关联维度
- 每个维度带来了多少增强分
- 涉及哪些关键线索

### 5.6 风险解释与核查要点区

该区进入“核查闭环”模块，需要把“分析结果”翻译成“核查语言”。

建议内容包括：

- 风险判断摘要
- 最应优先核查的区域
- 最应优先核查的对象
- 最应优先调取或复核的证据
- 建议现场核查问题清单

建议用“判断摘要 + 核查清单”结构，不做长篇论述。

### 5.7 最近任务与结果回流区

该区仍属于“核查闭环”模块，体现业务闭环，不让页面停留在分析结论。

建议展示：

- 已生成任务
- 当前任务状态
- 最近一次现场核查时间
- 核查结果摘要
- 回流后分值变化
- 风险等级变化
- 是否继续跟踪

本区要明确一个原则：

- 任务完成不等于风险自动消失
- 只有核查结论回流到线索有效性和评分依据后，才触发风险维持、升高、降低或解除

### 5.8 页内目录与模块控制原则

为避免页面再次变成“字太多、信息太密”的大盘式页面，建议增加以下控制原则：

- 左侧目录只承担页内定位，不承担跨页面导航
- 一级模块标题使用业务标题，不使用“为什么高、怎么查”这类说明式标题直接上屏
- “煤矿概况”只放结论，不放长清单
- “风险画像”只放支撑证据，不放全量任务
- “核查闭环”只放核查与回流，不重复大段评分说明
- 每个一级模块内部再拆 2 到 3 个子区即可，不并列过多主题
- 能用摘要卡表达的，不用大表格
- 能用 4 到 6 条重点项表达的，不展示 10 条以上明细

### 5.9 建议的页内目录结构

建议页内目录固定显示以下内容：

- 煤矿概况
- 风险画像
- 核查闭环

如需二级目录，可在对应模块下补充轻量子项：

- 煤矿概况
  - 综合评分
- 风险画像
  - 算法线索贡献
  - 关键线索
  - 关联增强分析
- 核查闭环
  - 核查要点
  - 任务与回流

## 六、字段口径

### 6.1 主对象口径

- 页面主对象是“煤矿”
- 不是“单条线索”
- 不是“单个算法”
- 不是“单次核查任务”

### 6.2 评分口径

综合风险分采用以下口径：

- 综合风险分 = 线索基础分 + 关联增强分 + 回流修正分

说明：

- 线索基础分：由算法权重、线索等级、可信度、时效性、有效性等形成
- 关联增强分：由时间、空间、对象、类型、状态冲突、历史复发等关系加权形成
- 回流修正分：由人工核查、专项排查、历史处置结果回流修正

### 6.3 线索口径

- 线索是评分输入和证据追溯对象
- 线索不是本页的主处置对象
- 本页只展示与当前煤矿风险解释最相关的关键线索

### 6.4 页面口径

本页必须始终围绕以下问题组织内容：

1. 为什么高
2. 高在哪里
3. 怎么查
4. 查完之后如何影响风险判断

## 七、交互要求

- 顶部固定系统导航保持现有框架，不在本页改动
- 页面仍在统一下钻舞台区内展示
- 页面应提供明确返回上一级专题页的入口
- 从关键线索、算法类别、任务记录进入下层细节时，应是轻量查看，不再继续无限下钻
- 不建议在本页再引入一个新的“大型综合研判台”式并列导航
- 页面允许纵向滚动，但不应表现为无引导的纯长页
- 建议通过左侧页内目录、当前模块高亮、清晰的内容分段，让用户自然从“煤矿概况”读到“风险画像”再读到“核查闭环”

## 八、视觉与表达要求

- 继续沿用 Av3 暗色驾驶舱风格
- 但本页比二级专题页更偏“单矿解释页”，应弱化驾驶舱大盘感，强化阅读节奏和信息分层
- 视觉重点应放在“评分构成”“关键证据”“核查建议”，而不是地图大屏或装饰性统计
- 页面要看起来像“煤矿画像”，不是“专题总览缩小版”
- 建议通过“左侧目录 + 右侧内容主体”的结构、段落留白和模块标题来降低密度
- 首屏优先做强结论感，而不是做满信息量

## 九、非本次范围

以下内容本次不在首版必做范围内：

- 真正接入接口和实时计算
- 所有线索的全量详情页
- 所有历史任务全量办理记录
- 跨专题煤矿画像复用模型
- 自动生成自然语言研判报告

## 十、当前建议的实现顺序

第一步先定结构：

- 先确认三级页模块结构和信息主线

第二步再定口径：

- 明确每一块展示什么字段、不展示什么字段

第三步再做视觉：

- 在 Av3 风格下把三级页做成“目录式煤矿画像页”而不是“又一个总览页”

第四步再进代码：

- 按三级页单独数据结构和页面组件实现

## 十一、验收标准

- 用户能一眼识别当前查看对象是哪座煤矿
- 用户能看懂综合风险分由哪些部分构成
- 用户能看懂哪些线索、哪些关联关系在支撑当前风险判断
- 用户能获得明确的现场核查重点
- 用户能看到最近任务和回流结果如何影响风险等级
- 页面阅读节奏清楚，用户不会在首屏就被大量信息压住
- 页面具备稳定的页内目录和模块定位，不会滚成一条无结构长页
- 页面与二级专题页职责清晰区分，不混成同一类页面
- 页面不再偏向此前误走的“综合研判台大工作台”方向

## 十二、当前状态

- 本文档为三级煤矿画像页的产品需求初版
- 当前仅用于收敛页面定位、结构和口径
- 尚未进入正式页面实现
