# 打非治违分级协同处置-专项研判产品迭代需求说明（2026-07-08）

## 一、迭代背景

蜀安首页 Av3 打非治违模块现有“分级协同处置”链路缺少第 3 页“专项研判 / 市级研判”。县级难以定性的复杂事项目前缺少统一承接页，导致多源线索归并、争议点沉淀、会商意见和处置去向无法在同一页面表达。

本轮补齐市级研判页，作为县级复核与省级挂牌督办之间的承接层，明确“复杂事项如何升级研判、如何形成意见、如何决定去向”。

## 二、迭代目标

- 在 Av3 drilldown 内补齐“专项研判 / 市级研判”页面，支持 hash route 直接访问。
- 让页面集中承载复杂事项主表、多源线索归并、异常时间轴和市级意见区。
- 保持与现有深蓝指挥舱样式一致，不引入新的全局壳层或导航体系。
- 让首页第三排“市级研判”卡片可最小接入到该页面。

## 三、迭代范围

本次迭代适用 `src/prototypes/system-replica-base/` 下打非治违 Av3 下钻体系。

### 3.1 涉及页面

| 页面 | 迭代类型 | 说明 |
| ---- | -------- | ---- |
| `shuan-home-command-v3-illegal-disposal-city-analysis` | 新增 drilldown 页面 | 分级协同处置第 3 页，专项研判 / 市级研判 |
| `shuan-home-command-v3` | 最小入口接入 | 首页第三排“市级研判”卡片点击进入新页面 |

## 四、需求说明

### 4.1 专项研判 / 市级研判页

#### 4.1.1 页面定位

- 页面角色：复杂事项升级研判页。
- 业务回答：
  - 县级不好定性的事项，市级如何做多源归并。
  - 市级如何沉淀争议点、形成研判意见并决定处置去向。
- 页面边界：只承载 drilldown 内容区，不承担全局导航或系统顶栏职责。

#### 4.1.2 页面结构

1. 页面标题条
   - 标题：`专项研判 / 市级研判`
   - 辅助说明：`复杂事项升级研判页`
   - 右侧操作：`退出`
2. 内容工具栏
   - 矿井 / 事项筛选
   - 统计周期筛选
   - 当前状态筛选
   - 刷新归并按钮
3. 顶部指标
   - 待研判
   - 研判中
   - 建议升级
   - 退回补查
   - 已形成意见
4. 左侧主区
   - 复杂事项主表格
   - 多源线索归并视图
   - 异常时间轴
5. 右侧意见区
   - 核心争议
   - 研判结论
   - 建议动作
   - 会商记录
6. 底部动作区
   - 维持整改
   - 继续核查
   - 转挂牌督办
   - 排除误报

#### 4.1.3 字段与交互

主表格字段固定为：

| 字段 | 说明 |
| ---- | ---- |
| 煤矿名称 | 当前事项所属煤矿 |
| 区域 | 所属市州 |
| 来源线索数 | 已归并的线索数量 |
| 关联专项 | 命中的专项整治类型 |
| 县级结论 | 县级上报结论 |
| 争议点 | 当前无法直接定性的核心冲突 |
| 当前状态 | 待会商 / 研判中 / 建议升级 / 退回补查 / 已形成意见 |

交互规则：

- 顶部退出按钮关闭当前 overlay，回到首页 Av3。
- 工具栏按钮本轮作为原型态表达，不接真实筛选逻辑。
- 首页“市级研判”卡片点击后打开本页。
- 底部动作按钮本轮仅表达处置去向，不接真实提交链路。

### 4.2 首页第三排“市级研判”卡片接入

#### 4.2.1 接入方式

- 保持首页原有四张“分级处置任务”卡片结构不变。
- 仅让“市级研判”卡片具备点击能力，打开 `shuan-home-command-v3-illegal-disposal-city-analysis`。
- 其他卡片继续保留静态展示，避免扩大共享冲突面。

## 五、交互与视觉要求

- 页面整体保持深蓝色指挥舱样式，复用现有卡片、按钮、分栏和边框语义。
- 页面顶部不得复制系统顶栏、模块导航或用户工具区。
- 工具栏、指标区、左右分栏和底部动作条在 1920 宽度下需完整展示。
- 中部“多源线索归并视图”需明确体现瓦斯 / 风 / 人 / 煤 / 电五类线索汇聚到单一复杂事项。
- 右侧意见区文案使用开发交付口径，避免写成聊天式说明。

## 六、验收标准

- `#page=shuan-home-command-v3-illegal-disposal-city-analysis` 可直接打开页面。
- 页面只占用 drilldown 内容区，不出现重复顶部导航。
- 页面包含 5 个顶部指标、主表格、多源归并、异常时间轴、意见区和 4 个底部动作。
- 首页第三排“市级研判”卡片点击后可打开该页面。
- 构建验证命令为：`$env:ENTRY_KEY='prototypes/system-replica-base'; npx vite build`
