# 2026-06-11 其他单位通讯录页面设计决策

## 背景

在 `system-replica-base` 原型的短信配置模块中，新增用于查看其他单位人员通讯录情况的页面。页面参考现有“政府接收人员”，但需要支持多个根节点单位，并把操作列调整为只读查看。

## 已确认需求

- 页面主要功能：展示其他单位人员的通讯录情况。
- 需要对比当前“政府接收人员”，但左侧通讯录树要同时展示多个根节点单位。
- 人员信息字段与政府接收人员接近。
- 字段调整：`短信接收规则` 改为 `短信接收规则数`。
- 不展示 `数据权限`。
- 操作列改为 `查看`。
- `查看` 不是空操作，需要复刻操作子页面。

## 设计决策

1. 在短信配置菜单下新增页面：`其他单位通讯录`。
2. 列表页沿用系统底座的左树右表结构：
   - 左侧为多根节点单位树，展示达州市应急管理局、大竹县应急管理局、宣汉县应急管理局等单位。
   - 右侧为查询区、工具区、人员表格。
3. 列表表格字段为：
   - 序号
   - 所属单位
   - 姓名
   - 职务
   - 手机号码
   - 短信接收规则数
   - 是否接收短信
   - 是否设置
   - 接收企业（家）
   - 操作：查看
4. 查看子页面为只读详情页：
   - 顶部提供返回列表。
   - 基本信息区展示所属单位、姓名、职务、手机号码、是否接收短信、是否设置、接收企业（家）、短信接收规则数。
   - 下方展示短信接收规则明细表，说明规则名称、报警类型、适用范围、启用状态。
5. 本轮不做新增、编辑、批量设置、数据权限配置。

## 验收重点

- 菜单中能进入“其他单位通讯录”。
- 列表左侧能看到多个根节点单位。
- 列表不展示数据权限。
- 列表字段展示“短信接收规则数”。
- 点击“查看”能进入子页面。
- 子页面展示人员信息和短信接收规则明细，并保持只读。

## 2026-06-11 补充：查看子页面按原系统配置页复刻

用户补充提供原政府接收人员“配置”操作页截图，确认查看页不能只做普通详情优化，需要复刻原操作页的信息结构。

本次补充决策：

- 查看子页面标题保持原系统口径：`配置`。
- 右上角使用系统一致的关闭 X，点击后返回列表。
- 保留原页面的 `基本信息` 分组，采用左右两列字段表格结构。
- 基本信息中展示姓名、职务、手机号码、是否接收短信、所属单位/部门、短信接收规则。
- 短信接收规则采用原系统标签堆叠形态，只读展示，不允许删除或编辑。
- 保留原页面的 `接收企业` 分组，包含企业名称选择框样式和企业表格。
- 底部 `保存 / 取消` 操作不展示，因为本页面是查看。
- 仍不展示 `数据权限` 字段。

## 2026-06-11 补充：查看页三段式只读结构

用户确认原系统短信接收规则的标签展示方式不适合作为查看页，需要拆解为独立表格，并保持不可编辑。

本次补充决策：

- 查看页固定拆为三个分组：`基本信息`、`短信接收规则`、`接收企业`。
- `基本信息` 只展示人员、单位、短信开关结果、规则数、接收企业数等摘要信息。
- `短信接收规则` 独立成表格，展示序号、规则名称、报警类型、规则级别、触发条件、接收范围、状态。
- `短信接收规则` 表格只读，不提供标签删除、选择、编辑、保存等操作。
- `接收企业` 独立成表格，只展示序号、区县、企业名称，不提供勾选框、企业选择框或保存操作。

## 2026-06-11 补充：查看页去操作和勾选

用户确认该页面是查看页，不应出现可操作语义。

本次补充决策：

- `短信接收规则` 表格字段按截图收敛为：规则名称、是否为煤矿规则、状态。
- 查看页内的 `短信接收规则` 表格不展示操作列。
- 查看页内的 `接收企业` 表格不展示勾选列，只保留序号、区县、企业名称。
- 三个分组右侧箭头作为真实收起/展开交互，点击后隐藏或展开对应内容。
