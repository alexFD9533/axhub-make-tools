# 多源线索总览页改版快照

日期：2026-07-06

范围：`src/prototypes/system-replica-base/pages/shuan-drilldowns/`

对应路由：`shuan-home-command-v3-illegal-algorithms`

## 本次确认的约束

1. Av3 下钻页继续使用统一框架：顶部固定菜单条 + 下方内容舞台。
2. 顶部固定菜单条仅由 `DrilldownFrame` 维护，子页面不得新增、复制或重写全局导航。
3. 本轮只改 `drill-stage` 内容区，不改首页 Av3 主体页面，不改 `ShuanHomeCommandV3.tsx`。
4. 下钻页面代码保持在 `src/prototypes/system-replica-base/pages/shuan-drilldowns/`。
5. 页面数据优先收敛在 `data.ts`，页面渲染维护在 `ShuanDrilldownPages.tsx`。
6. 页面样式继续使用 `style.css` 中的 `.drill-*` 命名，避免影响首页和其他模块。

## 本次设计决策

本页定位从“线索详情承载页”收敛为“多源线索总览工作台”，核心任务是帮助值守和监管人员快速判断：

- 哪些算法当前正在出线索。
- 这些线索主要集中在哪些矿井。
- 哪些算法和矿井需要优先关注。

页面结构确定为：

1. 顶部筛选区：时间、区域、矿井、状态等通用筛选。
2. 分类总览区：按“全部、管住瓦斯、管住风、管住人、管住煤、管住电、其他”展示总量和分类态势。
3. 左侧核心区“算法线索分布”：作为算法工作区，承载筛选、排序、分页和算法卡片矩阵。
4. 右侧辅助区上半部分“矿井线索集中排行”：突出线索聚集矿井。
5. 右侧辅助区下半部分“重点线索预览”：仅保留少量高价值线索预览，不展开完整明细表。

## 本次关键收敛

1. 不展示“四川添加算法 / 国家局算法”来源区分，统一按业务分类表达。
2. 不展示“线索综合分析”相关模块或分类入口。
3. 不保留“线索聚集研判”独立模块。
4. 原底部两个表格合并为“重点线索预览”。
5. 左侧主区保留卡片化表达，但改为“4 列卡片矩阵 + 排序 + 分页”的轻信息方案，承载大量算法。
6. 首页默认只看“有线索算法”，通过排序把最值得看的算法优先放到第一页。

## 分页结论

分页允许出现在“算法线索分布”模块头部右上角，作为模块内控件，不放到底部全页操作区。

分页承担的是“承载数量”的任务，不承担“表达结论”的任务，因此需要与以下能力同时存在：

- 分类页签
- 算法搜索
- 排序方式
- “只看有线索”开关
- 命中算法、线索数、命中矿井数、高风险数等总览指标

默认建议：

- 一行展示 4 个算法卡片，一屏展示 2 到 3 行。
- 每页展示 8 到 12 个算法卡片。
- 默认排序为“高风险优先”，其次按线索数、命中矿井数、今日新增排序。
- 分类切换后重新计算分页。

## 本轮不做

1. 不新增第二套顶部导航或页签系统。
2. 不改首页 Av3 卡片结构和点击入口。
3. 不展开三级页面，不新增接口接入。
4. 不在本页承载完整线索详情办理流程。
5. 不做跨页面联动状态持久化。
