# 蜀安首页 Av3 打非治违分级协同处置-现场核查页面决策快照

日期：2026-07-08

范围：`src/prototypes/system-replica-base/pages/shuan-drilldowns/`

对应路由：`shuan-home-command-v3-illegal-disposal-county-inspection`

## 本次确认的边界

1. 页面只实现蜀安首页 Av3 drilldown 覆盖层里的 `drill-stage` 内容区，不接管 `DrilldownFrame` 顶部壳层。
2. 页面顶部只保留“本页面标题 + 退出按钮”，筛选、搜索、催办等工具统一放入内容区 toolbox。
3. 保持 system-replica-base 现有深蓝指挥舱视觉语言，不新增独立系统顶栏、侧边导航或全局入口。
4. 优先把主体实现拆到独立局部组件，入口侧只做路由注册和最小点击接入。

## 页面定位

本页是打非治违“分级协同处置”的现场核查执行页，主要服务县级核查组、县局值班人员和上级督办人员，重点回答四个问题：

- 谁去查：当前责任组、负责人、同行人员、签到状态。
- 查了什么：线索来源、所属专项、核查对象、重点清单。
- 现场如何认定：现场记录、照片视频、GPS 轨迹、到离场时间。
- 下一步流向哪：属实 / 部分属实 / 不属实 / 转整改 / 转市级研判 / 建议挂牌督办。

## 结构决策

页面采用三段式结构：

1. 顶部任务态势区
   - 页面标题、退出按钮、筛选条件、搜索和催办操作。
   - 5 个核心指标：待签收、待核查、核查中、已回传、超时任务。
2. 中部执行主区
   - 左侧为核查任务清单表，承接煤矿名称、线索来源、所属专项、派发时间、截止时间、当前负责人、状态。
   - 右侧为当前执行摘要和下一步流向，直接表达“谁去查”和“认定后流向哪”。
3. 底部证据与认定区
   - 核查重点清单。
   - 现场记录。
   - 照片视频。
   - GPS / 轨迹与到离场。

2. 不接真实接口、不做批量操作闭环、不实现上传媒体交互，只用静态数据表达业务结构。
3. 不补市级研判、省挂牌督办的独立 drilldown 页面。
