# PRD-蜀安首页产品说明

# 模块说明

## 顶部系统栏

- 系统名称显示为：`蜀·安煤矿多源异构监测系统`。
- 中间放置系统主菜单，点击切换模块；
- 右边放置全局功能
  - 视频查看功能，点击后直接启动视频查看功能
  - 通知功能，通过声光模块通知报警、待办等信息
  - 人员账号设置
  - 人员姓名
  - 预留的系统快速设置、如白天/夜晚模式切换等，暂时不展示

## 左侧煤矿概况

左侧模块顺序为：

1. 煤矿生产状态统计。
2. 煤矿生产情况。
3. 从业人员情况。
4. 数据汇聚情况。
5. 煤矿瓦斯灾害情况、煤矿水害情况。

瓦斯灾害和水害作为自然灾害类补充信息，固定在左侧底部。

### 煤矿生产状态

按煤矿的生产状态展示煤矿数量数据

顶部展示煤矿总数

下方按类型展开，注意该类型非国家局通用类型字段，是四川自用的分类方式，后续需要开发业务让煤矿每天报告其生产状态

### 煤矿生产情况

展示区域煤矿的生产情况

展示字段内容为

* 本年度累计产煤量，数据来源为采掘通煤矿填报的产煤量按时间范围加和；根据煤矿填报实时更新

* 本月产煤量，同上

* 采煤工作面&掘进工作面，统计煤矿登记在采掘通的处于未完工状态的两类工作面的数量

### 从业人员情况

展示区域从业人员的基本情况

展示字段为

* 从业人员总数，展示区域里系统里登记的煤矿在职的人员总数，数据来源于从业人员管理

* 当前井下人员，展示区域煤矿所有在井下的人员总数，数据来源人员定位系统

* 井下带班领导&井下特种作业人员，展示区域煤矿所有在井下的相应职位/工种人员总数，数据来源人员定位系统

### 数据汇聚情况

 按类别展示系统汇聚了那些大项的数据。业务上数据来源于大数据平台，分项按项目文件的数据清单来；

### 煤矿瓦斯灾害情况&煤矿水害情况

展示地区煤矿在灾害项下的数量统计；数据来源于煤矿基本信息档案里的对应字段判断煤矿属于哪个级别，然后汇总统计

## 日常监管

按时间及区域范围展示以下子模块的业务数据

* 分级预警处置
- “危险作业报备 / 风险分级管控 / 隐患排查”。
- 主业务入口展示在“证照到期提醒 / 重大灾害提醒”之前。

### 分级预警处置

按100号文展示环境监测、人员定位、水害三个系统的分级报警

报警总数，汇总时间范围内三个系统产生的分级报警数量

预警分级分布，汇总时间范围内报警数据，根据级别分类展示报警数量

预警处置率，汇总时间范围内报警数据，根据煤矿处置情况计算处置率
环境监测、人员定位、水害，按系统汇总各类预警，顶部为系统汇总，下方为分类汇总，按类型数量排序，先排数量高的

数据来源，监测监控，人员定位、水害的预警后期通过算法实现

### 危险作业报备

按类型汇总时间内区域范围内煤矿报备项数量

展示报备数量最多2~3的数量

数据来源，报备系统

### 风险分级管控

按类型汇总时间内区域范围煤矿登记的风险数量

展示时间内新增的高风险（如没有则展示下一个登记，直到无新增风险）

展示时间内所有风险项有新动态登记的的数量

数据来源，双控—风险分级管控系统（待建设）

### 隐患排查

按类型汇总时间内区域范围煤矿登记的隐患数量

展示时间内新增的隐患数量

展示时间内所有待验的隐患数量

数据来源，双控—风险分级管控系统（待建设）

## 打非治违

按时间及区域汇总展示以下业务数据

1. 五类智能算法线索：瓦斯、风、人、煤、电。
2. 三类专项整治：隐蔽工作面专项整治、监控系统造假专项整治、隐瞒入井人数专项整治。
3. 分级处置任务：煤矿整改、县级走访、市级研判、省挂牌督办。

### 五类智能算法线索

汇聚时间范围内区域煤矿通过各类AI算法分析出来的问题线索数量

数据来源（矿数卫士）

### 三类专项整治

汇聚时间范围内区域煤矿通过矿数卫士选项分析算法分析出来的煤矿评级与工作安排情况

数据来源，矿数卫士，专题分析（待开发）

### 分级处置任务

汇集上游线索生成的预警和工作任务的处置情况

数据来源，矿数卫士，专题分析（待开发）
