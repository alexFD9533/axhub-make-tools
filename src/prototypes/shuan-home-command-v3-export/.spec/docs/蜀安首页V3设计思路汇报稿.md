# 蜀安首页 V3 设计思路汇报稿

## 一、设计定位

页面秉承首页及大屏的思路 ，以面向省级煤矿安全监管场景的“指挥台”为核心定位。

页面把分散在多个业务系统里的煤矿基础数据、生产运行数据、人员数据、监测预警数据和专项治理线索，集中组织成一个可扫视、可定位、可下钻的监管首页。

既满足所有人快速掌握全省煤矿的总体状态、了解相关工作的开展情况。同时也兼顾了一线人员快速了解核心业务版块情况，快速进入工作的需求。

## 二、设计要解决的问题

当前煤矿安全监管涉及的数据来源多、业务条线多、处置链条长。如果首页只是堆放功能入口，用户仍然需要进入多个模块反复查询，无法在值班、调度、研判时快速形成判断。

因此设计重点解决三个问题：

1. 全省情况能不能一眼看清：煤矿数量、生产状态、人员情况、灾害类型和数据汇聚情况要在首页形成总体画像。
2. 风险区域能不能快速定位：通过四川地图按市州聚合煤矿数量和风险等级，让监管人员先看到哪里更需要关注。
3. 业务处置能不能直接进入：日常监管和打非治违不只展示数字，还要保留预警处置、业务审核、专项整治和分级处置的入口。

## 三、整体布局思路

首页采用“左侧基础态势、中间地图聚合、右侧监管处置”的三栏结构。

左侧承担“底数和基础状态”的职责，展示煤矿生产状态、生产情况、从业人员、数据汇聚、瓦斯灾害和水害情况。这个区域帮助用户先建立监管对象的总体背景。

中间是地图主视觉，承担“区域定位和风险感知”的职责。四川地图按市州聚合煤矿数量，并使用红、橙、黄、蓝表达风险等级。顶部提供搜索定位，方便按区域或煤矿名称快速查找。

右侧承担“当日业务和处置闭环”的职责，上半部分是日常监管，下半部分是打非治违。这里把预警、审核、提醒、线索、专项整治和分级处置放在同一条业务线上，强调首页不是展示终点，而是进入工作的起点。

## 四、模块设计思路

### 1. 顶部系统栏

顶部保留系统名称、核心导航、视频入口、通知、用户和设置能力。设计上保持指挥舱风格，但不把顶部做成纯装饰，而是让它承担真实的全局导航和快捷入口。

导航项覆盖首页、日常监管、打非治违、从业人员管理、生产管理和系统设置，对应平台主要业务线。视频入口单独突出，是因为视频调度在煤矿监管场景中具有较高使用频率。

### 2. 左侧煤矿基础态势

左侧先展示煤矿生产状态统计，突出煤矿总数，并按生产矿井、建设矿井、隐患整改矿井、一停四不停矿井、井口封闭矿井拆分。这一层先回答“监管对象是什么状态”。

随后展示生产情况和从业人员情况，包括年度产煤量、本月产煤量、采掘工作面、从业人员总数、当前井下人员、带班领导和特种作业人员。这些指标对应监管中的生产活动和现场人员两个关键维度。

数据汇聚情况以六类数据卡片呈现，包括煤矿基础数据、人员与用工数据、自然灾害数据、安全监测数据、生产运行数据和事故保障数据。这样可以向汇报对象说明：平台首页不是孤立页面，而是建立在多源数据接入基础上的综合监管入口。

底部补充瓦斯灾害和水害情况，用于强化煤矿安全监管最关注的灾害类型，让首页既有生产状态，也有灾害背景。

### 3. 中间地图聚合

地图是页面的中心视觉，也是首页的主要研判区域。它以四川省域为背景，按市州聚合煤矿数量，并用风险颜色区分当前关注程度。

地图设计的重点不是做“大屏效果”，而是服务三个监管动作：看全省分布、看风险等级、快速定位区域或煤矿。搜索框支持按区域或煤矿名称模糊定位，地图点位支持进一步进入区域驾驶舱，后续可以扩展到市州、区县和具体煤矿层级。

### 4. 右侧日常监管

日常监管模块按照“总览、分类、入口、提醒”四段组织。

第一段展示预警总数、预警分级分布和预警处置率，先让用户知道当前监管压力有多大、风险等级如何、处置进展如何。

第二段按环境监测、人员定位、水害防治展示分级报警处置，聚焦当前最需要监管人员关注的预警来源。

第三段放置危险作业报备、风险分级管控、隐患排查三个业务入口，把首页从“看数据”延伸到“办业务”。

第四段展示证照到期提醒和重大灾害提醒，承担值班提醒和事项跟踪作用，避免重点事项淹没在大量指标中。

### 5. 右侧打非治违

打非治违模块围绕“发现线索、专项研判、分级处置”组织。

上层以瓦斯、风、人、煤、电五类智能算法线索作为入口，表达平台通过多源数据和算法发现异常线索的能力。

中层展示隐蔽工作面专项整治、监控系统造假专项整治、隐瞒入井人数专项整治，对应当前打非治违中的重点治理方向。

底层用煤矿整改、县级走访、市级研判、省挂牌督办展示分级处置链条，体现线索不是停留在发现层面，而是要进入闭环治理。

## 五、视觉表达思路

采用深色指挥舱风格，主要原因是该页面面向安全监管、值班调度和领导汇报场景，需要突出实时感、聚合感和风险感知。背景使用近黑蓝和深蓝，地图和面板使用青蓝描边，重点预警和关键数字使用琥珀黄、红、橙、黄、蓝等风险色。

视觉上控制装饰感，优先保证信息密度和可读性。面板结构清晰，数字层级突出，地图区域保持首屏中心位置，右侧业务模块保持完整闭环。整体表达是“监管指挥台”，而不是营销式大屏。

## 六、交互与扩展思路

首页保留了时间筛选、地图搜索、区域点位点击、更多入口和算法线索下钻等交互。当前原型重点展示首页组织方式，后续接入真实数据后，可以继续扩展四类能力：

1. 从省级地图下钻到市州、区县和具体煤矿。
2. 从预警统计下钻到预警列表、处置详情和闭环记录。
3. 从打非治违线索下钻到算法分析、证据链和处置任务。
4. 从数据汇聚情况下钻到数据源健康度、更新时间和异常数据治理。

## 七、汇报总结

系统首页的设计核心，是把首页从“功能导航页”升级为“安全监管指挥首页”。它用左侧基础态势回答“监管对象是什么情况”，用中间地图回答“风险集中在哪里”，用右侧监管和打非模块回答“今天要处理什么、线索如何闭环”。

通过这版首页，用户进入系统后可以先形成全省煤矿安全监管的总体判断，再根据风险区域、预警类型和专项线索进入具体业务处理。后续如果接入真实数据和下钻页面，这个首页可以继续作为省、市、县多级监管协同的统一入口。
