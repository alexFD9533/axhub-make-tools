# 短信管理模块产品迭代需求说明

> 项目：煤矿安全综合监控平台  
> 模块：短信管理 / 短信配置  
> 形成日期：2026-06-11  
> 来源会话：`019eb6d7-bd01-7730-a0c0-345b93de0c4d`、`019eb6de-699c-7c82-968b-b2aa834ba6be`  
> 原型底座：`src/prototypes/system-replica-base/`

## 一、迭代背景

本轮迭代围绕短信管理模块的两个关键页面展开：

1. **短信发送记录查询页**：原页面只能按本单位维度查看发送记录，无法区分和查询其他区县/单位的短信发送来源。
2. **其他单位通讯录页**：需要在现有“政府接收人员”能力基础上，增加一个面向其他单位人员通讯录的查看页面，用于监管侧查看多单位、多人员的短信接收配置情况。

本轮不另起新系统，不改变“煤矿安全综合监控平台”的整体底座，只在现有系统复刻原型中延展短信管理相关页面。

## 二、迭代目标

- 支持监管人员在短信发送记录中区分“本单位”和“其他单位”记录。
- 支持按发送单位查看其他单位短信发送来源。
- 在短信配置模块中新增“其他单位通讯录”，用于集中查看其他单位人员通讯录及短信接收配置情况。
- 将其他单位通讯录的人员查看页收敛为清晰的只读结构，避免出现编辑、保存、勾选等操作语义。
- 保持与现有平台后台样式、导航结构、页面容器和系统交互习惯一致。

## 四、需求说明

### 4.1 短信发送记录查询页改版

#### 4.1.1 页面定位

短信发送记录查询页用于查看平台短信发送记录。本次改版后，页面需要区分：

- **本单位**：查看当前单位自身发送记录。
- **其他单位**：查看其他区县/单位产生的短信发送记录。

#### 4.1.2 页面结构

页面标题保持为：**短信发送记录查询**。

查询区上方新增页签：

- 本单位
- 其他单位

默认进入页面时展示“本单位”。

#### 4.1.3 本单位页签

本单位页签沿用原页面查询和表格结构，保留原有字段与操作习惯。

验收要求：

- 默认展示本单位页签。
- 查询区字段与原页面保持一致。
- 表格仍展示本单位短信发送记录。

#### 4.1.4 其他单位页签

其他单位页签整体与本单位页签接近，但需要增加发送来源识别能力。

查询区新增字段：

| 字段 | 类型 | 示例值 |
|---|---|---|
| 发送单位 | 下拉选择 | 大竹县、达川区、宣汉县、开江县、渠县、万源市 |

表格新增字段：

| 字段 | 展示位置 | 说明 |
|---|---|---|
| 发送单位 | 序号之后 | 用于快速识别短信来源单位 |

验收要求：

- 点击“其他单位”页签后，查询区出现“发送单位”。
- “发送单位”下拉包含大竹县、达川区等区县单位。
- 表格中出现“发送单位”列，并展示对应模拟数据。
- 切回“本单位”后，页面恢复本单位字段结构。

### 4.2 新增其他单位通讯录页

#### 4.2.1 页面定位

其他单位通讯录用于查看其他单位人员的通讯录情况，参考现有“政府接收人员”页面，但需要支持多个根节点单位。

页面名称：**其他单位通讯录**  
建议入口：短信配置菜单下，与“企业接收人员”“政府接收人员”同级。

#### 4.2.2 列表页结构

采用现有政府接收人员类似的“左树右表”结构。

左侧：单位通讯录树，支持多个根节点单位，例如：

- 达州市应急管理局
- 大竹县应急管理局
- 宣汉县应急管理局
- 渠县应急管理局
- 万源市应急管理局

右侧：人员通讯录表，支持查询和查看。

#### 4.2.3 列表字段

其他单位通讯录列表字段如下：

| 字段 | 说明 |
|---|---|
| 序号 | 列表序号 |
| 所属单位 | 人员所在单位，便于脱离左树后仍能识别来源 |
| 姓名 | 人员姓名 |
| 职务 | 人员职务 |
| 手机号码 | 联系方式 |
| 短信接收规则数 | 原“短信接收规则”字段调整为数量口径 |
| 是否接收短信 | 展示是否开启短信接收 |
| 是否设置 | 展示是否完成相关配置 |
| 接收企业（家） | 展示关联接收企业数量 |
| 操作 | 仅展示“查看” |

明确不展示：

- 数据权限
- 新增
- 编辑
- 删除
- 批量配置

#### 4.2.4 查看入口

列表操作列展示为：**查看**。

点击“查看”后进入人员配置/查看子页面。该子页面不是空链接，也不是简单弹窗，需要复刻原政府接收人员配置页的信息结构，并按查看页口径去掉编辑能力。

### 4.3 其他单位通讯录查看页

#### 4.3.1 页面定位

查看页用于查看某个其他单位人员的短信接收配置情况。页面在视觉上参考原“政府接收人员”的配置页，但交互上必须是只读。

页面标题采用原系统口径：**配置**。

右上角关闭按钮使用系统一致的 **X**，点击后返回其他单位通讯录列表页。

#### 4.3.2 页面分组

查看页拆分为三个独立部分：

1. 基本信息
2. 短信接收规则
3. 接收企业

每个分组右侧有收起/展开箭头，点击后应真实收起或展开对应模块内容。

#### 4.3.3 基本信息

基本信息只展示摘要，不承载规则明细。

建议字段：

| 字段 | 说明 |
|---|---|
| 姓名 | 人员姓名 |
| 职务 | 当前职务 |
| 手机号码 | 联系方式 |
| 是否接收短信 | 是否接收短信通知 |
| 所属单位 | 人员所属单位 |
| 所属部门 | 人员所属部门或科室 |
| 短信接收规则数 | 关联规则数量 |
| 接收企业数 | 关联企业数量 |

要求：

- 所有字段只读展示。
- 不展示保存、取消、编辑按钮。
- 不展示数据权限。

#### 4.3.4 短信接收规则

原系统的规则标签堆叠方式不适合作为查看页展示方式，本次改为独立表格。

最终字段收敛为：

| 字段 | 说明 |
|---|---|
| 规则名称 | 短信接收规则名称 |
| 是否为煤矿规则 | 标识该规则是否属于煤矿规则 |
| 状态 | 启用/停用等状态 |

明确不展示：

- 操作列
- 编辑
- 删除
- 标签关闭
- 保存
- 选择控件

要求：

- 表格只读。
- 不出现任何可编辑或可删除语义。
- 与查看页定位保持一致。

#### 4.3.5 接收企业

接收企业作为独立模块，以表格只读展示。

字段为：

| 字段 | 说明 |
|---|---|
| 序号 | 企业序号 |
| 区县 | 企业所属区县 |
| 企业名称 | 企业名称 |

明确不展示：

- 勾选列
- 企业选择框
- 保存/取消
- 编辑/删除

要求：

- 表格只读。
- 不出现勾选框。
- 不出现可以修改接收企业范围的控件。

## 五、交互与视觉要求

### 5.1 视觉基调

- 沿用现有煤矿安全综合监控平台后台样式。
- 不引入新的视觉体系。
- 查询区、页签、表格、左侧树、分组标题、关闭按钮应与系统底座保持一致。

### 5.2 交互要求

| 交互 | 要求 |
|---|---|
| 页签切换 | 短信发送记录查询页可在本单位/其他单位之间切换 |
| 查看跳转 | 其他单位通讯录列表点击“查看”进入查看页 |
| 关闭返回 | 查看页右上角 X 返回列表页 |
| 分组折叠 | 基本信息、短信接收规则、接收企业右侧箭头可收起/展开 |
| 只读约束 | 查看页不得出现保存、取消、编辑、删除、勾选等操作语义 |

## 六、验收标准

### 6.1 短信发送记录查询

- 默认展示本单位页签。
- 其他单位页签可以切换。
- 其他单位页签中出现“发送单位”查询字段。
- 其他单位表格中出现“发送单位”列。
- 发送单位示例数据包含大竹县、达川区等。

### 6.2 其他单位通讯录列表

- 短信配置菜单下能进入“其他单位通讯录”。
- 左侧单位树能同时展示多个根节点单位。
- 表格字段包含“短信接收规则数”。
- 表格不展示“数据权限”。
- 操作列只展示“查看”。

### 6.3 其他单位通讯录查看页

- 点击“查看”能进入子页面。
- 页面标题为“配置”。
- 右上角关闭按钮为系统一致的 X。
- 页面拆分为基本信息、短信接收规则、接收企业三个部分。
- 短信接收规则表字段为规则名称、是否为煤矿规则、状态。
- 短信接收规则表不展示操作列。
- 接收企业表不展示勾选列。
- 三个模块右侧箭头可正常收起/展开。
- 页面整体只读，不出现保存、取消、编辑、删除等操作。
