## Page Intent

- Users: 煤矿安全监管人员、线索研判人员、执法检查组织人员
- Primary task or decision: 回答“这条煤量异常线索由哪些数据形成、数据之间为何矛盾、为什么判定为高风险，以及现场应核查什么”。
- Reading or operating path: 左侧先读异常事实和未解释煤量 → 中间查看六类数据如何印证 → 右侧读取证据构成、综合风险结论和现场核查事项 → 最后确认该线索的风险评分贡献。
- In scope: 单条“煤量比对异常”线索结论、异常窗口、关键煤量关系、六类多源印证、风险结论、现场核查事项、风险评分贡献、退出返回。
- Out of scope: 平台全局导航、专题风险研判、算法技术说明书、原始数据堆积、任务执行流程、关联任务入口、视频、人员轨迹、图纸对照、可信度说明、证据完整度、数据质量评分。

## Final Image-Generation Prompt

Use case: `ui-mockup`

生成一张约 1920 × 840 的超宽屏深蓝色煤矿安全监管 L3 单条线索分析页。页面主题为“煤量比对异常”，保留现有平台外部导航，因此图片只画内部标题栏：左侧“广元天安煤矿 - 煤量比对异常”，右侧“退出”。页面采用左中右三栏：左侧约 26% 展示异常情况、关键煤量和四柱对比图；中间约 51% 以 3 × 2 卡片呈现六类多源分析印证；右侧约 23% 按“证据构成 → 综合风险度 → 综合风险结论 → 现场核查事项 → 风险评分贡献”排列。

左侧使用清爽纵向信息层级，突出“未解释煤量 1,070 吨”和“较近期升高 76.3%”，四柱依次为近期产量基线 1,520、合法可解释煤量 1,610、主运输煤量 2,680、未解释煤量 1,070。中间六张卡分别为“提升运输运行时长”“提升运输用电量”“进尺工程测算”“入井人数匹配”“相对瓦斯涌出量匹配”“民爆品与采掘作业”，每张卡包含具体比较数据、一种适合自身逻辑的微型图表、一句业务结论和“支持异常”状态。瓦斯卡必须表达“相对瓦斯涌出量明显上涨、煤矿填报煤量基本未变，两项趋势不匹配，因此支持异常”。民爆品卡不得使用“火工品”。

右侧顶部使用三个并排图标摘要：“直接验证 3 项形成证据”“关联印证 3 项形成证据”“综合风险度 高”；其下是高风险结论；再下为六项现场核查事项；最底部展示“本线索风险评分贡献 +12.6 分”。删除综合可信度、可信度说明、证据完整度、数据质量和查看关联任务。证据摘要不是投票，不出现“6 票支持”。

视觉采用克制的政务监管深蓝风格：深 navy 背景、细青蓝边框、白色主文字、蓝灰辅助文字，橙色突出未解释煤量，红橙突出高风险，绿色仅用于“支持异常”。减少卡片套卡片、发光、装饰线和重复页脚；保持较充足留白和清晰阅读顺序。不得生成地图、视频、人员轨迹、图纸、任务流程、全局导航、营销页或 PPT 风格页面。

### Visual hierarchy

- Primary focus: 左侧“未解释煤量 1,070 吨”和四柱煤量对比。
- Secondary focus: 中间六张印证卡及其趋势差异；右侧“高风险”结论。
- Density and reading order: 横向三栏、栏内纵向阅读；先事实、再证据、后行动。中间使用两行三列卡片，但通过“直接印证/关联印证”分组避免普通六宫格观感。
- Typography: 建议使用“Microsoft YaHei / 思源黑体”；内部标题 20–22 px；区块标题 18–22 px；卡片标题 16–18 px；正文 13–15 px；关键数字 24–34 px；`+12.6分` 约 34–40 px。
- Color and elevation: 页面背景 `#031322`～`#061B2D`；面板背景 `#061C2F`；边框 `rgba(36,191,255,.42)`；主青 `#24BFFF`；辅助青 `#54D8FF`；支持绿 `#62D974`；异常橙 `#FF9F1C`；高风险红橙 `#FF584A`；主文字 `#F2F7FC`；次级文字 `#9FB6C8`。阴影和外发光保持低强度。
- Reusable visual patterns: 细线章节标题、无重度底色的指标行、单层证据卡、绿色状态胶囊、编号核查行、橙色评分贡献条。

- 不使用“6 票支持”或其他投票式风险表达。
- 不将本线索 `+12.6 分`误写为可信度。
- 不恢复“额外瓦斯涌出点”解释，也不设置“涌出点专项核查”。
- 瓦斯卡结论不得写成“不支持异常”。
- 不使用“火工品”，统一为“民爆品”。
- 不使用“观测值/基线值”作为六张证据卡的通用字段模板。
- 不增加全局导航、面包屑、平台名称、用户头像、时间栏、地图、视频、人员轨迹或图纸模块。
- 不做抽屉或弹窗布局；本页为完整内容页/全屏覆盖内容区。
- 不使用高亮紫色、过度发光、粗大箭头、卡片套卡片或大面积装饰空区。

## Implementation Acceptance

- Major-region tolerance or responsive behavior: 以 1867 × 842 为固定设计画布，按可用内容区等比缩放并居中；标题栏、三栏和六卡不得在桌面宽屏下重新换行。主要区域坐标偏差建议控制在 8–16 px（设计画布尺度）。
- Exact-copy requirements: 页面标题、六张证据卡标题、瓦斯结论、六项核查事项、“民爆品”、综合结论和“+12.6 分”必须逐字校验；不得复制参考图中的生成式错字。
- Required states/interactions: 退出返回 L2 上下文；数据缺失时不得继续显示无依据的“支持异常”；本页不出现任务执行控件。
- Screenshot comparison requirements: 在 1867 × 842 基准下，首屏应同时完整显示内部标题栏、左侧四柱图、中部六卡、右侧六项核查和底部评分贡献；不得出现纵向滚动条或内容裁切。
- Performance or accessibility constraints: 图表应使用轻量 SVG/Canvas；避免大量持续 `box-shadow` 动画；文字和背景对比度满足长时间监管查看；关键结论不能只依赖颜色表达，必须同时有文字状态。
