## Page Intent

- Users: 省、市、县煤矿安全监管人员，以及需要快速掌握全省打非治违态势的管理人员。
- Primary task or decision: 在约 10 秒内判断当前风险矿井规模和等级构成、当期任务执行情况、三个业务方面的风险分布，以及风险和任务的近期变化。
- Reading or operating path: 时间范围 → 风险矿井及等级构成 → 当期任务及状态 → 三个业务方面 → 四项趋势 → 进入同一个 L1 风险总览。
- In scope:
  - 风险矿井总数及中风险、较高风险、高风险构成；
  - 当期任务、当期新增、待办、执行中、复核中、当期闭环；
  - 隐蔽工作面、监控系统造假、隐瞒入井人数三个业务方面；
  - 风险矿井、新增/升级矿井、任务新增、任务闭环趋势；
  - 时间筛选和进入 L1 的点击入口。
- Out of scope:
  - 五类算法及算法数量；
  - 原始风险线索数量；
  - 县级任务、市级任务、省级任务卡片或“县级介入 / 市级介入 / 省级介入”文案；
  - 任务池操作、驻矿员分派、任务升级处理、人工发起入口；
  - 矿井排行、具体线索和证据链详情。

### Confirmed business relationships

1. 任务不是只有高风险才安排。风险达到中风险后即进入任务触发或监管介入范围。
2. 中风险、较高风险、高风险与县、市、省任务层级存在业务映射，但用户明确要求 L0 不展示“县级介入 / 市级介入 / 省级介入”字样。
3. 风险矿井和任务不是一一对应：
   - 一个任务可合并多个业务方面；
   - 同一任务可随风险变化动态升级；
   - 同一煤矿在不同周期可存在不同任务；
   - 当期任务包含跨期延续任务。
4. “当期任务”定义为任务生命周期与所选时间窗口存在交集的去重任务集合。
5. 当期任务四状态采用所选时段结束点的归类：待办、执行中、复核中、当期闭环互斥，并可加总得到当期任务总数。
6. “当期闭环”只统计闭环时间落在所选时间窗口内的任务。选择“今日”时，昨天已经闭环的任务不计入。
7. “当期新增”只统计任务生成并下发时间落在所选窗口内的任务，是补充事件指标，不与四状态互斥。
8. “风险矿井”是中性聚合概念，不等于“高风险矿井”，不得使用红色表达。

## Final Image-Generation Prompt

```text
设计“蜀安”煤矿多源异构监测系统 L0 首页右下固定区域的“打非治违重点风险”模块。只输出该模块，不输出全页。目标逻辑画布为 630×414 px，固定画布等比缩放；视觉需延续现有首页深 navy 蓝背景、青蓝技术线框、克制发光、小切角和紧凑中文数据大屏风格。

整体结构：
1. 顶部标题栏：左侧“打非治违重点风险”；右侧“今日 / 近7日 / 近30日”和“查看全部”。
2. 上半区左右两块：
   - 左侧为风险矿井总览。图标和“风险矿井”必须在同一水平行；放不下时缩小标题或图标，不换行、不扩大区域。下方大数字“47处”。右侧等级构成为“中风险 22处 / 较高风险 16处 / 高风险 9处”。风险矿井聚合指标使用中性青蓝/白色，绝不使用红色；红色只用于“高风险 9处”和高风险分段。
   - 右侧为“当期任务 55项”，辅助指标“当期新增 18项”，状态依次为“待办 19 / 执行中 15 / 复核中 9 / 当期闭环 12”，右上角显示“任务可合并、可升级”。
3. 中部三行：
   - 隐蔽工作面：风险等级分布 8/6/4，风险矿井 18处，当期新增/升级 +3处；
   - 监控系统造假：风险等级分布 7/5/3，风险矿井 15处，当期新增/升级 +2处；
   - 隐瞒入井人数：风险等级分布 7/5/2，风险矿井 14处，当期新增/升级 +2处。
   风险等级条使用绿色表示中风险、橙色表示较高风险、红色表示高风险。所有“风险矿井”数值使用青蓝/白色，不能用红色。
4. 底部四个等宽微型趋势卡：
   - 风险矿井 47处，青蓝色；
   - 新增/升级矿井 7处，橙色；
   - 任务新增 18项，蓝色；
   - 任务闭环 12项，绿色。
   每张卡显示稀疏、清晰的近30日趋势，末端日期为 07-15。

严格口径：煤矿数量统一使用“处”，任务数量使用“项”；不得出现“需介入矿井”“县级介入”“市级介入”“省级介入”；不得出现五类算法卡、原始风险线索数、县市省任务卡、矿井排行和任务管理入口。风险矿井不是告警等级，禁止用红色作为其主色。所有内容必须在 630×414 的真实区域内可读，不通过极端缩小字体硬塞内容。
```

## Negative Constraints

- Do not restore removed modules.
- 不得使用红色表达“风险矿井”聚合指标；红色只表示高风险等级及其分段。
- 左上图标和“风险矿井”必须在同一行；空间不足时缩小字号和图标，不得换行、堆叠或扩大模块。
- 不得出现“需介入矿井”或“需介入”。
- 不得出现“县级介入 / 市级介入 / 省级介入”。
- 不得使用“矿”作为煤矿数量单位，统一使用“处”。
- 不得恢复瓦斯、风、人、煤、电五类算法卡和算法数量。
- 不得展示原始风险线索数或以线索量替代风险程度。
- 不得恢复县级、市级、省级任务三卡。
- 不得增加任务管理、人工发起、任务升级操作入口。
- 不得增加矿井排行、地图、列表或证据链详情。
- 不得把该模块扩展成独立整页或改变右栏、日常监管、地图的尺寸。
- 不得将固定画布重排成移动端式单列布局。
- 不得使用普通浅色后台、营销落地页、PPT 流程图、大圆角 SaaS 卡片或过曝霓虹效果。
